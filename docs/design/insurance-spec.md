## 概要

このドキュメントは、保険商品推奨ポータルの**機能要件を実装可能な仕様に変換したもの**です。  
Phase 1 の要件定義書（`insurance-recommendation-requirements.md`）をベースに、設計・実装に必要な詳細仕様を記載します。

## 詳細機能仕様

### M1: 顧客情報管理

#### 1.1 会員登録フロー

**ユースケース**: UCM1-001 ユーザー新規登録

**フロー:**
```
1. ランディングページ表示
2. 「推奨診断を始める」ボタンクリック
3. 登録方法選択（メール/Google/LINE）
4. メールアドレス入力 → 確認メール送信
5. 確認メール内リンク → プロフィール入力フォーム
   - 氏名
   - 年齢（必須）
   - 性別（オプション）
   - 電話番号（オプション）
6. 基本情報確認画面
7. 登録完了 → ホーム画面へ遷移
```

**エラーハンドリング:**
- 重複メール：「既に登録済みです。ログインしてください」
- 無効なメール形式：「正しいメール形式を入力してください」
- タイムアウト：「確認メールが30分の有効期限切れです。再度メール送信してください」

**データベーススキーマ:**
```sql
-- users テーブル
CREATE TABLE users (
  id UUID PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  name VARCHAR(100),
  age INT,
  gender ENUM('M', 'F', 'OTHER'),
  phone VARCHAR(20),
  auth_method ENUM('EMAIL', 'GOOGLE', 'LINE'),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 1.2 プロフィール管理

**入力項目:**

| 項目 | 型 | 必須 | 備考 |
|------|-----|---|----|
| 氏名 | TEXT | ✓ | |
| 年齢 | INT | ✓ | 18-99 |
| 性別 | ENUM | - | M/F/OTHER |
| 電話番号 | PHONE | - | SMS送信用 |
| 職業分類 | ENUM | ✓ | 会社員/自営業/パート/学生/無職 |
| 家族構成 | COMPOSITE | - | 配偶者/子ども人数 |
| 居住地（都道府県） | ENUM | - | 災害リスク判定用 |
| 既存保険有無 | BOOLEAN | - | 見直し要否判定 |

### M2: ヒアリングエンジン - 損害保険

#### 2.1 自動車保険フロー

**フロー構成:**
```
ステップ1: 保有車両状況
  ├─ 車を持っていますか?
  │  ├─ はい → ステップ2へ
  │  └─ いいえ → 「自動車保険は不要です」→ 火災保険へ
  │
ステップ2: 運転状況（単一選択）
  ├─ 毎日運転する
  ├─ 週に数回運転する
  ├─ 月に1回程度運転する
  └─ ほぼ運転しない（駐車のみ）
  │
ステップ3: 年間走行距離（単一選択）
  ├─ 5,000km以下（短距離）
  ├─ 5,000～10,000km（通常）
  ├─ 10,000～20,000km（多走行）
  └─ 20,000km以上（超多走行）
  │
ステップ4: 車両タイプ（複数選択 or 単一）
  ├─ 普通乗用車
  ├─ 軽自動車
  ├─ タクシー・運送業（商用）
  └─ バイク（二輪車）
  │
ステップ5: 過去事故歴（複数選択可）
  ├─ 事故なし
  ├─ 物損事故（3年以内）
  ├─ 人身事故（3年以内）
  ├─ 自損事故
  └─ その他
  │
ステップ6: 運転者管理（複数選択可）
  ├─ 本人のみ
  ├─ 配偶者あり
  ├─ 子ども（18-25歳）あり
  └─ その他家族
  │
ステップ7: 補償内容ニーズ（複数選択可）
  ├─ 対人賠償（対向車運転手への補償）
  ├─ 対物賠償（相手方物への補償）
  ├─ 車両保険（自分の車修理）
  ├─ 搭乗者傷害
  ├─ ロードサービス（JAF相当）
  └─ その他特約
  │
ステップ8: 保険料予算（単一選択）
  ├─ 月3,000円以下
  ├─ 月3,000～5,000円
  ├─ 月5,000～10,000円
  ├─ 月10,000円以上
  └─ 予算なし（最適保険を提案）
  │
ステップ9: 確認・レビュー画面
  └─ 「推奨を見る」ボタン → マッチング実行
```

**スコアリングロジック:**

```javascript
// 自動車保険スコアリング例
const autoInsuranceScore = (responses) => {
  let score = 0;
  
  // 高リスク要因（加点）
  if (responses.drivingFrequency === 'daily') score += 30;
  if (responses.annualMileage >= 20000) score += 20;
  if (responses.pastAccidents.length > 0) score += 15 * responses.pastAccidents.length;
  if (responses.drivers.includes('child')) score += 25;
  
  // 補償ニーズ（判定）
  if (responses.needsVehicleInsurance) score += 25;
  
  // バジェット制約（調整）
  if (responses.budget === 'under3000') score *= 0.7;
  
  return Math.min(score, 100);
};
```

#### 2.2 火災保険フロー

**フロー構成:**
```
ステップ1: 対象物選択（排他的単一選択）
  ├─ 戸建住宅（一軒家）
  ├─ マンション（分譲）
  ├─ 賃貸物件（マンション・アパート）
  ├─ テナント事業所
  └─ 住宅なし（対象なし）

  ├─ 家財のみ
  └─ 建物 + 家財
  │
ステップ3: 主な災害リスク（複数選択可）
  ├─ 豪雨・洪水
  ├─ 雹（ひょう）
  ├─ 雪害
  ├─ 地震
  ├─ 落雷
  └─ 近隣火災
  │
ステップ4: 特約ニーズ（複数選択可）
  ├─ 盗難補償
  ├─ 水漏れ補償（上階からの水漏れ）
  ├─ 破損・汚損補償

ステップ5: 居住地（都道府県）
```

### M3: マッチングエンジン

#### 3.1 保険会社マッチング

**マッチング対象保険会社（損害保険）:**

| ID | 会社名 | 強み | 対応保険種 |
|----|-------|------|---------|
| IC001 | 三井住友海上 | 業界大手、安定性 | 自動車、火災、賠償 |
| IC002 | あいおいニッセイ同和 | 代理店ネットワーク | 自動車、火災、傷害 |
| IC003 | 東京海上日動 | 事故対応評価 | 自動車、火災、特約充実 |
| IC004 | 損保ジャパン | 特約充実 | 自動車、火災、健康増進 |
| IC005 | Sony損保 | オンライン割引 | 自動車（ダイレクト） |

**マッチングアルゴリズム:**

```
1. 顧客スコアと保険会社の強みを照合
2. 優先度スコア計算
   score = (顧客ニーズマッチ度 × 0.4) + (保険会社評判スコア × 0.3) + (保険料競争力 × 0.3)
3. スコアでランキング（1位〜3位）
4. 根拠テキスト生成
```

#### 3.2 根拠生成

**例1: 自動車保険（多走行ドライバー）**
```
推奨理由：
「あなたのような高走行ドライバーには、【三井住友海上】の
『トータルアシスト自動車保険』がおすすめです。

理由：
・年間走行距離 20,000km 以上の顧客向けの割引制度
・24時間対応ロードサービス
・過去事故実績による割引適用可能」
```
