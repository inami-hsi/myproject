## 1. 概要

### 1.1 プロジェクトサマリー

TaskFlowは個人およびスモールチーム（最大10名）向けのタスク管理Webアプリケーションである。ガントチャート・カンバンボード・カレンダービューを統合し、タスクの可視化とスケジュール管理を直感的に行える。

| 項目 | 値 |
|------|-----|
| アプリ名 | TaskFlow |
| 種別 | タスク管理アプリ（ガントチャート付き） |
| 想定利用者 | 最大10名 |
| フレームワーク | Next.js 14+ (App Router) |
| ORM | Prisma 6.0+ (PostgreSQL) |
| 状態管理 | Zustand |
| UIライブラリ | shadcn/ui + Tailwind CSS |
| 認証 | CC-Auth (Cognito) |

### 1.2 機能スコープ

**In Scope:**
- タスクCRUD（サブタスク含む）
- ガントチャートビュー（日/週/月切替、依存関係矢印、今日線、ドラッグ&ドロップ）
- カンバンボードビュー（ステータス別カラム、D&D移動）
- カレンダービュー（月/週表示）
- リストビュー（ソート・フィルタ対応）
- プロジェクト管理（複数プロジェクト、アーカイブ）
- マイルストーン管理
- タスク依存関係（FS/SS/FF/SF）
- タグ・カテゴリ管理
- コメント機能（Markdown対応）
- 進捗管理（0-100%スライダー）
- 期限管理・通知
- 検索・フィルタリング
- エクスポート（CSV/JSON）
- ダークモード
- レスポンシブ対応

**Out of Scope:**
- チャット機能
- ファイル添付
- ワークフロー自動化
- 外部サービス連携（Slack, Jira等）

### 1.3 技術アーキテクチャ概要

```
[Browser] <--HTTPS--> [Next.js App Router (ECS Fargate)]
                              |
                       [API Routes]
                              |
                       [Prisma ORM]
                              |
                       [PostgreSQL (RDS)]

状態管理: Zustand (Client-side)
認証: CC-Auth / Cognito (JWT)
CDN: CloudFront
DNS: Route53
CI/CD: CodeBuild + CodePipeline
```

#### ベースURL

```
/api
```

#### 認証

全APIエンドポイントは認証済みユーザーのコンテキストで動作する。現在の実装では `getOrCreateDemoUser()` によるデモユーザーを使用。CC-Auth統合後はCognito JWTトークンによる認証に切り替える。

```
Content-Type: application/json
```

```json
{
  "error": "エラーメッセージ（日本語）",
  "details": {
    "fieldErrors": {},
    "formErrors": []
  }
}
```

#### HTTPステータスコード

| コード | 意味 | 使用場面 |
|--------|------|---------|
| 200 | OK | 取得・更新成功 |
| 201 | Created | 作成成功 |
| 400 | Bad Request | バリデーションエラー、必須パラメータ不足 |
| 404 | Not Found | リソースが見つからない |
| 409 | Conflict | 重複エラー（例: タグ名の重複） |
| 500 | Internal Server Error | サーバー内部エラー |

#### POST /api/tasks

新規タスクを作成する。

**リクエストボディ:**

```typescript
type Request = {
  title: string;              // 必須、1文字以上
  description?: string;       // 任意
  status?: TaskStatus;        // 任意、デフォルト: "TODO"
  priority?: Priority;        // 任意、デフォルト: "MEDIUM"
  startDate?: string;         // 任意、ISO 8601 datetime
  endDate?: string;           // 任意、ISO 8601 datetime
  dueDate?: string;           // 任意、ISO 8601 datetime
  projectId: string;          // 必須、1文字以上
  parentId?: string;          // 任意、サブタスク用

};
```

**バリデーションルール:**

| フィールド | ルール |
|-----------|-------|
| title | 必須、string、min(1) |
| description | 任意、string |
| status | 任意、enum: TODO / IN_PROGRESS / DONE / ON_HOLD |
| priority | 任意、enum: CRITICAL / HIGH / MEDIUM / LOW |
| startDate | 任意、ISO 8601 datetime |
| endDate | 任意、ISO 8601 datetime |
| dueDate | 任意、ISO 8601 datetime |
| projectId | 必須、string、min(1) |
| parentId | 任意、string |
| tagIds | 任意、string[] |

1. ユーザーのプロジェクト所有権を検証
2. sortOrderの最大値を取得し +1 を設定

4. tagIdsが指定されている場合、TagOnTask中間テーブルにレコード作成

**レスポンス:** `201 Created` - 作成されたタスクオブジェクト（tags, dependencies, dependents含む）

**エラーレスポンス:**

| コード | エラー |
|--------|-------|
| 400 | `"バリデーションエラー"` + details |
| 404 | `"プロジェクトが見つかりません"` |
| 500 | `"タスクの作成に失敗しました"` |

## 7. エラーハンドリング方針

### 7.1 エラー分類

| カテゴリ | HTTPコード | ハンドリング方法 |
|---------|-----------|----------------|
| バリデーションエラー | 400 | フォームフィールド横にインラインエラー表示 |
| 認証エラー | 401 | /login へリダイレクト |
| 認可エラー | 403 | エラートースト + 前画面へ戻る |
| リソース未存在 | 404 | エラートースト or 404ページ |
| 重複エラー | 409 | インラインエラー（「既に存在します」） |
| サーバーエラー | 500 | エラートースト（「エラーが発生しました。再試行してください」） |
| ネットワークエラー | - | エラートースト + リトライボタン |

### 7.2 バリデーション

**サーバーサイド（Zod）:**

| リソース | スキーマ | 主要ルール |
|---------|---------|----------|
| Project (create) | createProjectSchema | name: min(1), color: hex6 |
| Project (update) | updateProjectSchema | name: min(1) optional, archived: boolean optional |
| Task (create) | createTaskSchema | title: min(1), projectId: min(1), date: ISO 8601 |
| Task (update) | updateTaskSchema | progress: 0-100, date: nullable ISO 8601 |
| Comment (create) | createCommentSchema | content: min(1) |
| Milestone (create) | createMilestoneSchema | name: min(1), date: ISO 8601, projectId: min(1) |
| Milestone (update) | updateMilestoneSchema | name: min(1) optional, date: ISO 8601 optional |
| Tag (create) | createTagSchema | name: min(1) unique, color: hex6 optional |

**クライアントサイド:**
- フォーム送信前にZodスキーマで事前バリデーション
- リアルタイムバリデーション（onChange / onBlur）
- 送信ボタンはバリデーションエラー時にdisabled

### 7.3 エラーUI表示

| コンポーネント | 用途 |
|-------------|------|
| Toast (shadcn/Sonner) | 一時的なエラー/成功通知 |
| Inline Error | フォームフィールド直下のバリデーションエラー |
