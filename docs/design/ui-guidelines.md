## 1. Aesthetic Direction

### 1.1 Direction: calm-functional

TaskFlowのUIは「穏やかで機能的」を基本方針とする。毎日長時間使うツールとして、派手さよりも「邪魔をしない」ことを最優先とし、ユーザーがタスクの内容そのものに集中できる静かなインターフェースを実現する。

### 1.2 Mood: professional

穏やかで集中力を高める空間。視覚ノイズを最小化し、タスクの内容に注意を向けさせる。色数を抑え、余白を十分に確保することで「呼吸感」のあるUIを設計する。

### 1.3 References

| サイト | 参考ポイント |
|--------|-------------|
| Linear | ミニマルUI、キーボードショートカット中心の操作、余白設計 |

## 2. Differentiation

> **穏やかな色調と十分な余白で「呼吸感」のあるガントチャート。**

従来のガントチャートツール（MS Project, Asana, Monday.com等）が持つ「情報密度の高さ = 使いにくさ」の問題を排除する。タスクの流れを直感的に理解でき、開いた瞬間に「今何をすべきか」が分かるデザイン。

**Tagline**: 「静かで心地よいガントチャート」

### Design Thinking

1. **Purpose** -- ガントチャートとカンバンでタスクの全体像と進捗を可視化する
2. **Tone** -- calm-functional。余白と静寂が主役
3. **Constraints** -- WCAG AA準拠、Lighthouse Performance 90+、200ms以下のインタラクション
4. **Differentiation** -- 「開いた瞬間に今日やることが分かる」静かなUI。ガントチャートの情報密度を下げ、呼吸感を生む

## 3. Typography Guidelines

### 3.1 Font Families

| 用途 | フォント | フォールバック | 選定理由 |
|------|---------|---------------|---------|
| 見出し (H1-H4) | **DM Sans** | system-ui, sans-serif | 幾何学的でありながら温かみがある。Interの代替として個性的 |
| 本文 | **Source Serif 4** | Georgia, serif | 読みやすいセリフ体。長文の可読性が高い |
| コード・数値 | **JetBrains Mono** | monospace | 等幅で視認性が高い。進捗率や日付表示に最適 |

### 3.2 Font Scale

| 要素 | フォント | ウェイト | サイズ | 行間 |
|------|---------|---------|--------|------|
| H1 | DM Sans | 700 (Bold) | 2.5rem (40px) | 1.2 |
| H2 | DM Sans | 600 (SemiBold) | 2rem (32px) | 1.25 |
| H3 | DM Sans | 600 (SemiBold) | 1.5rem (24px) | 1.3 |
| H4 | DM Sans | 500 (Medium) | 1.25rem (20px) | 1.4 |
| Body | Source Serif 4 | 400 (Regular) | 1rem (16px) | 1.6 |
| Body Large | Source Serif 4 | 400 (Regular) | 1.125rem (18px) | 1.6 |
| Caption | DM Sans | 400 (Regular) | 0.875rem (14px) | 1.4 |
| Code | JetBrains Mono | 400 (Regular) | 0.875rem (14px) | 1.5 |
| Gantt Label | DM Sans | 500 (Medium) | 0.75rem (12px) | 1.2 |

### 3.3 Typography Rules

- **`text-balance`** を見出し（H1-H4）に適用する。行末の不自然な改行を防ぎ、見出しの視覚的安定性を高める
- **`tabular-nums`** を数値表示（進捗率、日付、カウンター）に適用する。数字の幅が揃い、値の変化時にレイアウトシフトが発生しない
- **`max-w-prose`** (65ch) を本文テキストに適用する。最適な可読性を保つ行長を維持する
- 行間は本文で **1.6**、見出しで **1.2-1.4** を守る
- セリフ体（Source Serif 4）は本文・説明文に限定し、UIラベルやボタンにはサンセリフ体（DM Sans）を使用する

### 3.4 Prohibited Fonts

以下のフォントは**絶対に使用しない**。

| フォント | 禁止理由 |
|---------|---------|
| **Inter** | AIが最も多く選択するフォント。個性がない |
| **Roboto** | AIが最も多く選択するフォント。個性がない |
| **Arial** | AIが最も多く選択するフォント。個性がない |
| **Space Grotesk** | AI生成アプリの典型フォント |

## 4. Color Guidelines

### 4.1 Principles

- **支配的な色と鋭いアクセント**: 深いネイビー (`#1a1a2e`) をベースに、テラコッタオレンジ (`#e07a5f`) で意図的に視線を誘導する
- 色数を最小限に抑え、各色に明確な意味を持たせる
- アクセントカラーはCTAとアクティブ状態に限定使用し、乱用しない
- ステータスを表す色（success, warning, danger）は意味と1:1で対応させる

### 4.2 Main Colors

| 用途 | カラー | 使用箇所 |
|------|--------|---------|
| Primary `#1a1a2e` | 深いネイビー | ヘッダー、サイドバー |
| Secondary `#f8f8f2` | ソフトホワイト | カード背景 |
| Accent `#e07a5f` | テラコッタオレンジ | CTA、ガントバー（進行中）、フォーカスリング |
| Accent 2 `#6d8b74` | セージグリーン | 完了状態、成功 |
| Accent 3 `#8b7ec8` | ラベンダー | マイルストーン |

### 4.3 Background & Text

| 用途 | Light | Dark |
|------|-------|------|
| Background | `#fafaf8` | `#16161a` |
| Card Background | `#f8f8f2` | `#1e1e24` |
| Text Primary | `#2d2d2d` | `#e4e4e7` |
| Text Muted | `#9ca3af` | `#71717a` |
| Border | `#e5e5e5` | `#27272a` |

### 4.4 Status Colors

| Status | Color | 用途 |
|--------|-------|------|
| Warning | `#d4a843` | 期限警告（3日以内） |
| Danger | `#c44e4e` | 期限超過、エラー |
| Success | `#6d8b74` | 完了、成功 |
| Info | `#6b9ac4` | インフォメーション |

### 4.5 Contrast Rules

- ミュートテキスト (`#9ca3af`) とアクセントカラー (`#e07a5f`) は**大テキスト（18px以上 or 14px Bold以上）でのみ使用**する
- 小テキストではプライマリテキストカラー (`#2d2d2d` / `#e4e4e7`) を使用する
- ダークモードでアクセントカラーのコントラスト比が不足する場合は明度を10%上げる

### 4.6 Dark Mode

- **必須対応**。システム設定連動 + 手動トグルの両方をサポート
- 設定は `localStorage` に永続化する
- ガントチャートのダークモード対応は必須
- ダークモードでもWCAG AA準拠のコントラスト比を維持する

## 5. Motion Guidelines

### 5.1 Principles

1. 全てのインタラクションアニメーションは **200ms以下**
2. **Compositor props のみ**使用: `transform`, `opacity`
3. レイアウトプロパティ（`width`, `height`, `top`, `left`）のアニメーションは**禁止**
4. **`prefers-reduced-motion`** を尊重する。`@media (prefers-reduced-motion: reduce)` でアニメーションを無効化する

### 5.2 Animation Library

**motion/react** を使用する。Framer Motionの後継ライブラリであり、React Serverコンポーネント対応。

### 5.3 Animation Definitions

| 操作 | Duration | Easing | Properties |
|------|----------|--------|------------|
| ボタンホバー | 150ms | ease-out | opacity, transform(scale) |
| カード展開 | 200ms | ease-out | opacity, transform |
| モーダル表示 | 200ms | ease-out | opacity, transform(translateY) |
| サイドバー開閉 | 200ms | ease-in-out | transform(translateX) |
| ガントスケール切替 | 200ms | ease-out | transform(scaleX) |
| ドロップダウン | 150ms | ease-out | opacity, transform(translateY) |
| トースト通知 | 200ms | ease-out | opacity, transform(translateX) |
| ページ遷移 | 150ms | ease-in-out | opacity |

### 5.4 Implementation Pattern

```tsx
import { motion } from "motion/react";

// Correct: compositor props only
<motion.div
  initial={{ opacity: 0, y: 8 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.2, ease: "easeOut" }}
>
  {children}
</motion.div>

// WRONG: layout property animation
// <motion.div animate={{ width: "100%" }} /> // PROHIBITED
```

### 5.5 Reduced Motion

```tsx
import { useReducedMotion } from "motion/react";

function AnimatedComponent() {
  const shouldReduceMotion = useReducedMotion();

  return (
    <motion.div
      animate={shouldReduceMotion ? {} : { opacity: 1, y: 0 }}
      transition={shouldReduceMotion ? { duration: 0 } : { duration: 0.2 }}
    />
  );
}
```

## 6. Layout Principles

### 6.1 Grid System

**8px Grid System** を採用する。全てのスペーシングは8の倍数を基本とする。

| Token | Value | 用途 |
|-------|-------|------|
| xs | 4px | インライン要素間 |
| sm | 8px | 要素内パディング（小） |
| md | 16px | 要素内パディング（標準） |
| lg | 24px | セクション間マージン |
| xl | 32px | セクション間マージン（大） |
| 2xl | 48px | ページセクション間 |
| 3xl | 64px | ヒーロー領域 |

### 6.2 Whitespace Philosophy

- 余白は「無駄」ではなく「呼吸」。要素間に十分な余白を確保する
- カード内のパディングは `md` (16px) 以上を基本とする
- セクション間は `lg` (24px) 以上の余白を確保する
- ガントチャートのバー間余白は `sm` (8px)

### 6.3 Border Radius Mapping

| 要素 | 角丸 | Tailwind Class |
|------|------|---------------|
| ボタン | 8px | `rounded-lg` |
| カード | 12px | `rounded-xl` |
| モーダル | 16px | `rounded-2xl` |
| ガントバー | 6px | `rounded-md` |
| 入力フィールド | 8px | `rounded-lg` |
| アバター | 9999px | `rounded-full` |
| タグ/バッジ | 9999px | `rounded-full` |

### 6.4 Shadows

シャドウは**控えめに**使用する。カードは `shadow-sm` を基本とし、ホバー時に `shadow-md` へ遷移する。

| Level | Value | 用途 |
|-------|-------|------|
| sm | `0 1px 2px rgba(0, 0, 0, 0.04)` | カード、ガントバー |
| md | `0 4px 6px rgba(0, 0, 0, 0.07)` | ホバー状態、ドロップダウン |
| lg | `0 10px 15px rgba(0, 0, 0, 0.1)` | モーダル、ポップオーバー |

### 6.5 Z-Index Scale

一貫したz-indexスケールを使用してレイヤーの重なりを管理する。

| Layer | z-index | 用途 |
|-------|---------|------|
| Base | 0 | 通常のコンテンツ |
| Gantt Today Line | 10 | 今日線 |
| Sticky Header | 20 | ガントタイムラインヘッダー |
| Sidebar | 30 | サイドバー |
| Dropdown | 40 | ドロップダウンメニュー |
| Modal Backdrop | 50 | モーダル背景 |
| Modal | 51 | モーダル本体 |
| Toast | 60 | トースト通知 |
| Tooltip | 70 | ツールチップ |

### 6.6 Height Strategy

- ページ全体のレイアウトには **`h-dvh`** (Dynamic Viewport Height) を使用する
- `h-screen` ではなく `h-dvh` を使用することで、モバイルブラウザのアドレスバー表示/非表示時のちらつきを防ぐ

## 7. Icon Guidelines

### 7.1 Library

**Lucide React** を使用する。統一感のあるストロークアイコン。

### 7.2 Size Scale

| 用途 | Size | Tailwind Class |
|------|------|---------------|
| インラインアイコン | 16px | `w-4 h-4` |
| ボタンアイコン | 20px | `w-5 h-5` |
| ナビゲーション | 24px | `w-6 h-6` |
| 空状態イラスト | 48px | `w-12 h-12` |

### 7.3 Stroke Width

デフォルトの `strokeWidth={2}` を使用する。**カスタマイズ禁止**。

### 7.4 Color Independence

アイコンのみで意味を伝えてはならない。テキストラベルまたは `aria-label` を必ず併用する。色覚多様性への配慮として、色のみに依存しない情報設計を行う。

## 8. Gantt Chart Specific Guidelines

### 8.1 Bar Visual Design

| Status | Bar Color | Background (Unfilled) | Pattern |
|--------|-----------|----------------------|---------|
| Not Started | `#d1d5db` | 同色 | Solid |
| In Progress | `#e07a5f` | `rgba(224, 122, 95, 0.2)` | 進捗率でfill |
| Completed | `#6d8b74` | 同色 | 全体塗りつぶし |
| On Hold | `#9ca3af` | -- | 斜線パターン |

- 角丸: 6px (`rounded-md`)
- 高さ: Desktop 28px / Touch 36px
- バー間余白: 8px

### 8.2 Progress Label

- バー幅 80px 以上の場合にインライン表示
- Font: DM Sans 500, 12px
- 色: バー色に対するコントラスト確保（白またはダークテキスト）

### 8.3 Dependency Arrows

- Default: `#9ca3af`, 1px solid
- Hover: `#e07a5f`, 2px solid, 200ms transition
- Arrow Head: triangle, 6px

### 8.4 Today Line

- Color: `#c44e4e` (dashed, 2px dash / 4px gap)
- Label: "today" (top position)

### 8.5 Milestone

- Shape: Diamond (45deg rotated square), 16px
- Color: `#8b7ec8` (Lavender)
- Label: milestone name (right side)

### 8.6 Timescale

- Modes: Day / Week / Month
- Transition: 200ms ease-out
- Header: 2-tier (upper: Month/Year, lower: Day/Week number)
- Weekend columns: slightly darker background (`#f5f5f3` light / `#1e1e22` dark)

### 8.7 Drag Interaction

- Ghost preview at 50% opacity during drag
- Snap to day grid
- Resize cursor: `col-resize` on bar edges
- Tooltip feedback showing new dates during drag

## 9. Anti-Patterns (NEVER)

以下のパターンは TaskFlow のデザインにおいて**絶対に使用しない**。

### 9.1 Fonts

| PROHIBITED | Reason |
|-----------|--------|
| Inter | AI slop -- AIが最も多く選択するジェネリックフォント |
| Roboto | AI slop -- 個性がない |
| Arial | AI slop -- 個性がない |
| Space Grotesk | AI生成アプリの典型フォント |

### 9.2 Colors

| PROHIBITED | Reason |
|-----------|--------|
| 紫グラデーション on 白背景 | AI生成UIの典型配色 |

### 9.3 Layout

| PROHIBITED | Reason |
|-----------|--------|
| 予測可能なレイアウトパターン（Hero + 3カラムグリッド等の定型パターン） | テンプレート感のある没個性デザイン |

### 9.4 Effects

| PROHIBITED | Reason |
|-----------|--------|
| 過剰なドロップシャドウ（全カードに `shadow-xl` を適用する等） | AI生成の典型パターン |
| グロー効果（ネオンのような発光効果） | AI slop |
| Cookie-cutter design（テンプレート感のある没個性デザイン） | AI slop |

### 9.5 Shadows

| PROHIBITED | Reason |
|-----------|--------|
| `shadow-xl` の常時適用 | 過剰なシャドウは AI slop |
| `shadow-2xl` の常時適用 | 過剰なシャドウは AI slop |

### 9.6 Motion

| PROHIBITED | Reason |
|-----------|--------|
| `will-change` の常時適用 | パフォーマンス影響。必要な時のみ |
| `backdrop-filter: blur()` の過度な使用 | パフォーマンス影響 |
| 300ms超のアニメーション | 過剰な演出 |
| bounce, elastic等の過度なイージング | 過剰な演出 |

### 9.7 Gradients

| PROHIBITED | Reason |
|-----------|--------|
| グラデーション背景の多用 | AI生成の典型パターン。TaskFlowはソリッドカラーを基本とする |
