## 2. Design Principles

### 2.1 Radix UI Primitives

全てのインタラクティブコンポーネントは Radix UI プリミティブをベースとする。アクセシビリティ（キーボード操作、スクリーンリーダー、フォーカス管理）が標準で組み込まれている。

### 2.2 cn() Utility

`clsx` + `tailwind-merge` による**クラス名結合ユーティリティ**を全コンポーネントで使用する。

```typescript
// lib/utils.ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

**使用例**:

```tsx
import { cn } from "@/lib/utils";

function Card({ className, ...props }: React.ComponentProps<"div">) {
  return (
    <div
      className={cn(
        "rounded-xl bg-secondary shadow-sm border border-border p-4",
        className
      )}
      {...props}
    />
  );
}
```

### 2.3 Compound Component Pattern

コンポーネントは Compound Component パターンで設計し、柔軟な構成を可能にする。

```tsx
// Compound Component example
<Card>
  <Card.Header>
    <Card.Title>Task Name</Card.Title>
    <Card.Description>Due in 3 days</Card.Description>
  </Card.Header>
  <Card.Content>
    <ProgressBar value={65} />
  </Card.Content>
  <Card.Footer>
    <Badge variant="priority-high">P1</Badge>
  </Card.Footer>
</Card>
```

### 2.4 Accessibility Requirements

- **aria-label**: 全インタラクティブ要素に必須
- **aria-describedby**: 補足説明がある場合に使用
- **role**: 非標準要素に適切なロールを付与
- **keyboard navigation**: Tab, Enter, Escape, Arrow keys をサポート
- **focus indicator**: 全フォーカス可能要素に明確なフォーカスリングを表示

```tsx
// Correct: aria-label on interactive element
<Button aria-label="Create new task">
  <Plus className="w-5 h-5" />
</Button>

// Correct: accessible icon button
<IconButton
  aria-label="Delete task"
  icon={<Trash2 className="w-5 h-5" />}
/>
```

## 3. shadcn/ui Base Components

### 3.1 Button

**Base**: shadcn/ui Button
**Customization**: テラコッタアクセント (`#e07a5f`)

| Variant | Description | Use Case |
|---------|-------------|----------|
| `default` | テラコッタ背景 + 白テキスト | 主要CTA |
| `destructive` | Danger赤背景 | 削除操作 |
| `outline` | ボーダーのみ | 副次アクション |
| `secondary` | セカンダリ背景 | 補助ボタン |
| `ghost` | 背景なし | ツールバーアクション |
| `link` | テキストリンクスタイル | インラインリンク |

| Size | Height | Padding | Font Size |
|------|--------|---------|-----------|
| `sm` | 32px | px-3 | 14px |

**Spec**: [component-specs/button.md](./component-specs/button.md)

## 4. Custom Components

### 4.1 GanttChart

ガントチャート本体コンポーネント。タスクの時間軸表示を担当する。

| Prop | Type | Description |
|------|------|-------------|
| `tasks` | `Task[]` | タスク配列 |
| `dependencies` | `Dependency[]` | 依存関係配列 |
| `viewMode` | `"day" \| "week" \| "month"` | タイムスケールモード |
| `onTaskUpdate` | `(task: Task) => void` | タスク更新コールバック |
| `onTaskClick` | `(task: Task) => void` | タスククリックコールバック |
| `showMinimap` | `boolean` | ミニマップ表示（xl+ のみ） |

**Structure**:

```
<GanttChart>
  <GanttTimeline />        <!-- タイムラインヘッダー -->
  <GanttRows>
    <GanttBar />           <!-- 各タスクバー -->
    <GanttDependencyArrow /> <!-- 依存関係矢印 -->
    <GanttMilestone />     <!-- マイルストーン -->
  </GanttRows>
  <GanttTodayLine />       <!-- 今日線 -->
</GanttChart>
```

## 5. Component Composition Rules

```
src/
  components/
    ui/           # shadcn/ui base components
      button.tsx
      card.tsx
      dialog.tsx
      input.tsx
      select.tsx
      badge.tsx
      sheet.tsx
      tabs.tsx
      progress.tsx
      slider.tsx
      tooltip.tsx

      separator.tsx
      dropdown-menu.tsx
    gantt/         # Gantt chart components
      gantt-chart.tsx
      gantt-bar.tsx
      gantt-timeline.tsx
      gantt-dependency-arrow.tsx
      gantt-milestone.tsx
      gantt-today-line.tsx
    kanban/        # Kanban board components
      kanban-board.tsx
      kanban-column.tsx
      kanban-card.tsx
    task/          # Task-specific components
      task-card.tsx
      task-form.tsx

```

### 5.2 Import Convention

```tsx
// shadcn/ui components
import { Button } from "@/components/ui/button";
import { Card, CardHeader, CardTitle, CardContent } from "@/components/ui/card";

// Custom components
import { GanttChart } from "@/components/gantt/gantt-chart";
import { KanbanBoard } from "@/components/kanban/kanban-board";

// Utilities
import { cn } from "@/lib/utils";
```

### 5.3 Variant Definition Pattern

cva (class-variance-authority) を使用してバリアントを定義する。

```tsx
import { cva, type VariantProps } from "class-variance-authority";

const buttonVariants = cva(
  "inline-flex items-center justify-center rounded-lg font-medium font-heading transition-colors focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-accent/20 disabled:pointer-events-none disabled:opacity-50",
  {
    variants: {
      variant: {
        default: "bg-accent text-white hover:bg-accent/90",
        destructive: "bg-danger text-white hover:bg-danger/90",
        outline: "border border-border bg-transparent hover:bg-secondary",
        secondary: "bg-secondary text-primary hover:bg-secondary/80",
        ghost: "hover:bg-secondary",
        link: "text-accent underline-offset-4 hover:underline",
      },
      size: {
        default: "h-10 px-4 text-sm",
        sm: "h-8 px-3 text-sm",
        lg: "h-12 px-6 text-base",
        icon: "h-10 w-10",
      },
    },
    defaultVariants: {
      variant: "default",
      size: "default",
    },
  }
);
```
