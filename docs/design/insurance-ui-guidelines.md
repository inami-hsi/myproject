## 概要

本ドキュメントは、保険商品推奨ポータルのUI/UX設計の基本方針を定めます。  
`insurance-design-system.yml` をベースに、具体的な画面設計・インタラクション・アクセシビリティ要件を記載します。

## 1. デザイン美学

### 1.1 ブランドアイデンティティ

```
Tone:      trust-simplicity（信頼性 × シンプルさ）
Mood:      professional-friendly（プロフェッショナル × 親しみやすさ）
Tagline:   「複雑な保険選びをシンプルに」
Vision:    「段階的な訴求 → わかりやすい結果」
```

### 1.2 ターゲットペルソナ別デザイン

| ペルソナ | 年代 | 優先度 | デザイン対応 |
|---------|------|--------|-----------|
| **P1: オンライン自動加入派** | 25-40 | 速さ・シンプルさ | モバイルUI最適化、最小クリック化 |
| **P2: 対面相談志向派** | 40-60 | 詳細さ・信頼感 | 詳細説明パネル、専門家紹介 |
| **P3: 比較検討派** | 30-50 | 比較表・説得力 | 比較表機能、根拠明示 |

### 1.3 デザイン原則（DO / DON'T）

#### DO: 実装すべき

✓ **視覚的優先度の明確化**  
　→ 推奨1位（1st rank）は目立つ緑色、2-3位は淡い色
  
✓ **段階的フロー**  
　→ 1ページ1アクション、ユーザーの認知負荷削減
  
✓ **根拠の透明性**  
　→ 「なぜこの保険会社か」を必ず記載（ロ方式準拠）
  
✓ **心理的安心感**  
　→ 保険業法準拠表示、専門家サポート訴求
  
✓ **モバイルファースト**  
　→ スマートフォンから設計、PC対応へ拡張

#### DON'T: 禁止事項

✗ **不確実な推奨**  
　→ スコア根拠なしの推奨は掲載しない
  
✗ **視覚的混乱**  
　→ 同じランク（2位・3位）の色の区別化は控える
  
✗ **過度なアニメーション**  
　→ ページ読込時刻を損なわないよう200ms以内に制限
  
✗ **複数一気説明**  
　→ 1ページで複数商品の詳細説明をしない
  
✗ **信頼感の低下要因**  
　→ 不透明な手数料、非表示のプライバシー項目

### 4.1 スペーシングスケール

基本単位: **8px**

```

8px  (sm)  ← ボタン内パディング最小
16px (md)  ← セクション内スペース（標準）
24px (lg)  ← セクション間隔（中）
```

## 6. アクセシビリティ（A11y）

### 6.1 WCAG AA準拠項目

**コントラスト比**
```
通常テキスト:   4.5:1以上（WCAG AA）
大きいテキスト: 3:1以上
```

例：
- テキスト色 #333333 on 背景 #FFFFFF ✓ (12.6:1)
- テキスト色 #666666 on 背景 #FFFFFF ✓ (5.12:1)

**キーボード操作**
```
・すべてのボタン・フォーム要素は Tab キーでフォーカス可能
・フォーカス状態は視覚的に明確（枠線または背景色の変更）
・フォーカス順序は論理的（左→右、上→下）
```

**タッチターゲット**
```
・ボタン・リンク最小サイズ: 44px × 44px
・ボタン間隔: 最小 8px
```

**スクリーンリーダー対応**
```
・img要素に alt テキスト必須
・aria-label を使用した説明（例: aria-label="推奨理由を表示"）
・role="alert" を使用したエラーメッセージ
```

### 6.2 色覚バリアフリー

✓ 色のみに頼らない（テキスト + アイコン併用）
✓ 色覚異常シミュレーション確認済み

例：
- ランキング: 色 + 数字（「1位」「2位」「3位」）
- バッジ: 色 + テキスト（「推奨」）

## 7. アニメーション・モーション

### 7.1 アニメーション原則

**タイミング**
```
インタラクション:  200ms以下（ボタンクリック → フィードバック）
遷移/トランジション: 300-500ms
ページ読込:         Compositor properties のみ
```

**許可されるプロパティ**
```
✓ transform (translate, scale, rotate)
✓ opacity
```

**禁止されるプロパティ**
```
✗ height, width （Jank / 再レイアウト原因）
✗ top, left, right, bottom
✗ margin, padding（必要に応じて transform を代替）
```

### 7.2 インタラクション例

**ボタンホバー**
```
動作: 背景色変更 + 軽いシャドウ
期間: 200ms
イージング: ease-out
```

**質問パネル展開**
```
動作: max-height or scale transform
期間: 300ms
イージング: ease-in-out
```
