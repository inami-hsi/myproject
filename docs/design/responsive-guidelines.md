## 1. Approach: Mobile First

全てのスタイルはモバイル（最小画面幅）から記述し、ブレークポイントで段階的に拡張する。Tailwind CSS の responsive prefix (`sm:`, `md:`, `lg:`, `xl:`, `2xl:`) を使用する。

```tsx
// Correct: Mobile first
<div className="p-4 md:p-6 lg:p-8">
  <h1 className="text-xl md:text-2xl lg:text-[2.5rem]">Project Name</h1>
</div>

// WRONG: Desktop first (min-width overrides)
// <div className="p-8 max-md:p-4"> -- DO NOT
```

## 3. Layout Strategy

### 3.1 Overview

| Breakpoint | Sidebar | Gantt Chart | Kanban | Main Content |
|-----------|---------|-------------|--------|-------------|
| < md | 非表示 (hamburger) | 横スクロール | 1 column vertical | Full width |
| md - lg | 折りたたみ可能 | 横スクロール | 2 columns | Full width - sidebar |
| lg - xl | 固定表示 (240px) | フル表示 | 4 columns | calc(100% - 240px) |
| xl+ | 固定表示 (280px) | フル表示 + ミニマップ | 4 columns | calc(100% - 280px) |

### 3.2 Sidebar

```
Mobile (< md):
┌──────────────────┐
│  Header [=]      │  <- ハンバーガーメニュー
├──────────────────┤
│                  │
│  Main Content    │
│                  │
└──────────────────┘

Tablet (md - lg):
┌────┬─────────────┐
│ [<]│             │  <- 折りたたみトグル
│ 60 │   Main      │
│ px │  Content    │
│    │             │
└────┴─────────────┘
  or
┌─────────┬────────┐
│ 240px   │        │  <- 展開時
│ Sidebar │  Main  │
│         │Content │
└─────────┴────────┘

Desktop (lg+):
┌──────────┬───────────────────┐
│  240px   │                   │
│ Sidebar  │   Main Content    │  <- 常時表示
│  (fixed) │                   │
└──────────┴───────────────────┘

Wide (xl+):
┌───────────┬──────────────────────┐
│  280px    │                      │
│ Sidebar   │   Main Content       │
│  (fixed)  │                      │
└───────────┴──────────────────────┘
```

**Implementation**:

```tsx
<aside
  className={cn(
    // Mobile: hidden by default, shown as overlay
    "fixed inset-y-0 left-0 z-30 w-[280px] transform transition-transform duration-200 ease-in-out",
    "md:relative md:translate-x-0",
    // Tablet: collapsible
    "md:w-[60px] md:data-[expanded=true]:w-[240px]",
    // Desktop: always visible
    "lg:w-[240px]",
    // Wide: wider
    "xl:w-[280px]",
    // Mobile: off-screen unless open
    isOpen ? "translate-x-0" : "-translate-x-full md:translate-x-0"
  )}
>
  {/* Sidebar content */}
</aside>
```

### 3.3 Gantt Chart

| Breakpoint | Behavior |
|-----------|----------|
| < md | 横スクロール。タスク名は左固定パネル（120px）。バー高さ 36px（タッチ対応） |
| md - lg | 横スクロール。タスク名パネル 180px。バー高さ 28px |
| lg - xl | フル表示。タスク名パネル 240px。バー高さ 28px |
| xl+ | フル表示 + ミニマップ。タスク名パネル 280px |

**Gantt Bar Height**:

```tsx
<GanttBar
  className={cn(
    "h-[36px]",         // Mobile: touch-friendly
    "md:h-[28px]"       // Desktop: compact
  )}
/>
```

**Horizontal Scroll**:

```tsx
<ScrollArea className="w-full" orientation="horizontal">
  <div className="min-w-[800px] lg:min-w-0">
    <GanttChart />
  </div>
</ScrollArea>
```

### 3.4 Kanban Board

| Breakpoint | Columns | Card Width |
|-----------|---------|------------|
| < md | 1 column (vertical scroll) | Full width |
| md - lg | 2 columns | 50% |
| lg+ | 4 columns (status-based) | 25% |

**Implementation**:

```tsx
<div
  className={cn(
    "flex flex-col gap-4",           // Mobile: vertical stack
    "md:grid md:grid-cols-2 md:gap-4",  // Tablet: 2 columns
    "lg:grid-cols-4"                    // Desktop: 4 columns
  )}
>
  <KanbanColumn status="not_started" />
  <KanbanColumn status="in_progress" />
  <KanbanColumn status="completed" />
  <KanbanColumn status="on_hold" />
</div>
```

### 3.5 View Switcher (Tabs)

ガント / カンバン / リストの切替はレスポンシブに対応する。

| Breakpoint | Behavior |
|-----------|----------|
| < md | ボトムナビゲーション or セグメントコントロール |
| md+ | タブバー (Tabs component) |

## 4. Touch Interaction

### 4.1 Touch Target Size

全てのインタラクティブ要素は最小 **44px x 44px** のタッチターゲットを確保する。

```tsx
// Correct: 44px minimum touch target
<button className="min-h-[44px] min-w-[44px] p-2">
  <Plus className="w-5 h-5" />
</button>

// WRONG: too small
// <button className="p-1"><Plus className="w-4 h-4" /></button>
```

### 4.2 Gantt Bar Drag (dnd-kit)

- **dnd-kit** を使用してタッチ対応のドラッグを実装する
- モバイルではバー高さを 36px に拡大しドラッグしやすくする
- ドラッグ中はゴーストプレビュー（50% opacity）を表示する
- 日単位グリッドにスナップする
- ドラッグ中に新しい日程をツールチップで表示する

```tsx
import {
  DndContext,
  useDraggable,
  useDroppable,
  TouchSensor,
  PointerSensor,
  useSensor,
  useSensors,
} from "@dnd-kit/core";

const sensors = useSensors(
  useSensor(PointerSensor, {
    activationConstraint: { distance: 5 },
  }),
  useSensor(TouchSensor, {
    activationConstraint: { delay: 150, tolerance: 5 },
  })
);
```

### 4.3 Pinch Zoom

ガントチャートのタイムスケール変更にピンチズームを対応する。

| Gesture | Action |
|---------|--------|
| Pinch in | タイムスケール縮小 (month -> week -> day) |
| Pinch out | タイムスケール拡大 (day -> week -> month) |

タイムスケール切替のアニメーション: 200ms ease-out

### 4.4 Swipe

| Component | Swipe Direction | Action |
|-----------|----------------|--------|
| Kanban Board | Left / Right | カラム間スクロール（モバイル） |
| Task Detail Sheet | Right | シートを閉じる |
| Sidebar (mobile) | Left | サイドバーを閉じる |

### 4.5 Long Press

| Component | Action |
|-----------|--------|
| Gantt Bar | コンテキストメニュー表示 |
| Kanban Card | ドラッグモード開始 |
| Task Row | マルチセレクトモード |

## 5. Safe Area

### 5.1 iOS Notch / Dynamic Island

`env(safe-area-inset-*)` を使用してノッチやDynamic Islandに対応する。

```css
/* Safe area padding */
.app-container {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}
```

### 5.2 Viewport Height

**`h-dvh`** (Dynamic Viewport Height) を使用する。`h-screen` は使用しない。

```tsx
// Correct: Dynamic Viewport Height
<main className="h-dvh flex flex-col">
  <Header />
  <div className="flex-1 overflow-hidden">
    <GanttChart />
  </div>
</main>

// WRONG: h-screen causes issues on mobile browsers
// <main className="h-screen"> -- DO NOT
```

`h-dvh` はモバイルブラウザのアドレスバー表示/非表示時の高さ変動に追従し、ちらつきを防ぐ。

## 6. Container Queries

### 6.1 Purpose

コンポーネント単位でレスポンシブ制御を行うために Container Queries を活用する。親コンテナの幅に基づいてスタイルを切り替えることで、コンポーネントが配置される場所に依存しない設計を実現する。

### 6.2 Use Cases

| Component | Container Query | Behavior |
|-----------|----------------|----------|
| TaskCard | `@container (min-width: 300px)` | コンパクト表示 -> フル表示 |
| GanttBar progress label | `@container (min-width: 80px)` | ラベル表示 / 非表示 |
| KanbanCard | `@container (min-width: 250px)` | 詳細情報の表示切替 |

### 6.3 Implementation

```tsx
// Parent: define container
<div className="@container">
  <TaskCard />
</div>

// Child: respond to container width
<div className={cn(
  "flex flex-col gap-1",
  "@[300px]:flex-row @[300px]:items-center @[300px]:gap-4"
)}>
  <span className="font-heading font-medium">{task.title}</span>
  <Badge className="hidden @[300px]:inline-flex">{task.priority}</Badge>
</div>
```

### 6.4 Tailwind Configuration

```typescript
// tailwind.config.ts
{
  plugins: [
    require("@tailwindcss/container-queries"),
  ],
}
```

## 9. Performance Considerations

### 9.1 Image Optimization

- `<Image>` component (Next.js) を使用し、自動的にレスポンシブ画像を生成する
- アバター画像は 32px / 40px / 48px の3サイズを用意する
- `loading="lazy"` をファーストビュー外の画像に適用する

### 9.2 Code Splitting

- ガントチャート、カンバンボードは `React.lazy` + `Suspense` で遅延読み込みする
- ビュー切替時に必要なコンポーネントのみをロードする

```tsx
const GanttChart = lazy(() => import("@/components/gantt/gantt-chart"));
const KanbanBoard = lazy(() => import("@/components/kanban/kanban-board"));

<Suspense fallback={<ViewSkeleton />}>
  {view === "gantt" && <GanttChart />}
  {view === "kanban" && <KanbanBoard />}
</Suspense>
```

### 9.3 Virtualization

- ガントチャートのタスク行はビューポート内のみレンダリングする（仮想スクロール）
- カンバンボードのカードも長いリストの場合は仮想化する
- `@tanstack/react-virtual` の使用を推奨

### 9.4 Critical CSS

- ファーストビューに必要なスタイルのみを初回レンダリングで適用する
- Tailwind CSS の purge 設定で不要なスタイルを削除する
