## 1. 美的方向性（Aesthetic Direction）

### 1.1 トーン

- [x] **brutally-minimal** - 厳格なミニマリズム、極端なネガティブスペース

企業検索ツールはデータが主役。UIは徹底的に削ぎ落とし、
検索条件とデータテーブルにフォーカスさせる。
不要な装飾を排除し、情報密度と操作効率を最大化する。

### 1.2 ムード

- [x] **professional**（プロフェッショナル）
- [x] **industrial**（インダストリアル）

ビジネスツールとしての実用性を最優先。信頼感とデータの正確性を伝えるUI。

### 1.3 参照サイト

| サイト名 | URL | 参考ポイント |
|---------|-----|-------------|
| BIZMAPS | https://biz-maps.com | 企業検索UIの標準パターン |
| Crunchbase | https://www.crunchbase.com | データテーブルの情報密度と操作性 |
| gBizINFO | https://info.gbiz.go.jp | 政府データの検索UI・地域フィルタ |
| Airtable | https://airtable.com | テーブルビューのUX |

### 1.4 禁止パターン（AI Slop回避）

以下は**使用禁止**:

- [x] Inter, Roboto, Arial等のジェネリックフォント
- [x] 紫グラデーション on 白背景
- [x] 予測可能なレイアウトパターン
- [x] 過剰なドロップシャドウ
- [x] Space Grotesk（AI生成の典型）
- [x] cookie-cutter design
- [x] 過度な装飾・イラスト・ヒーローイメージ

### 1.5 差別化ポイント

```
「検索条件を選ぶだけ。3クリックでリスト完成。」
業種ツリーと地域マップの直感的なフィルタUIで、
専門知識がなくても誰でも企業リストを作れる体験。
検索結果の件数がリアルタイムに変化するライブカウンターで
絞り込みの過程を可視化する。
```

- ツール系UIでの過度な演出（バウンス、スプリング等）

- prefers-reduced-motion 非考慮

## 8. 追加要件

### 8.1 ライブカウンター

- フィルタ条件変更時に検索結果件数をリアルタイム更新
- 件数は tabular-nums + 等幅フォントで桁揃え
- 件数変化時に微細な数字カウントアップアニメーション

### 8.2 データテーブル

- TanStack Table によるソート・ページネーション
- 列幅リサイズ対応
- カラム表示/非表示の切替
- 行クリックで企業詳細モーダル
- コンパクト表示モード（行高を小さく）

### 8.3 フィルタUI

- 業種は階層ツリー（大分類→中分類→小分類の展開式）
- 地域は都道府県→市区町村のカスケード選択
- 選択中の条件はフィルタチップ（pill型バッジ）で上部に表示
- 全条件クリアボタン

### 8.4 ダウンロードUI

- ダウンロード形式（CSV/Excel）と文字コード（UTF-8/Shift-JIS）の選択
- 残りダウンロード件数のプログレスバー
- 大量件数（5,000件超）は非同期生成 → メール通知

### 8.5 メールデザイン

- react-email でHTML/テキスト両対応
- ブランドカラー統一
- 新規法人通知メールはリスト形式で企業名・業種・所在地を簡潔に表示
