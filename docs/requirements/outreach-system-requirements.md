# アウトリーチシステム 要件定義書

## 1. プロジェクト概要

### 1.1 プロダクト名
**Outreach Engine**（仮称）

### 1.2 概要
メール一斉配信と企業ホームページの問い合わせフォームへの自動営業を統合したBtoBアウトリーチシステム。CSVインポートしたメールリストへの一斉/予約/ステップメール配信と、企業サイトの問い合わせフォームを自動検出・自動送信するフォーム営業機能を提供する。

### 1.3 ターゲットユーザー
- **自社利用**: 営業チームのアウトリーチ効率化
- **外部提供(SaaS)**: 中小企業の営業・マーケティング担当者

### 1.4 プロジェクト構成
- **独立プロジェクト**として新規構築
- 既存の企業リストビルダー（company-list-builder）のデータと連携可能

## 2. 機能要件

### 2.1 メール配信機能

#### 2.1.1 リスト管理
| 機能 | 詳細 |
|------|------|
| CSVインポート | メールアドレス、会社名、担当者名、カスタムフィールドを一括取り込み |
| 手動追加 | 個別にコンタクト情報を登録 |
| リスト管理 | 複数リストの作成・編集・削除・マージ |
| 重複排除 | インポート時に同一メールアドレスの重複を自動検出・除外 |
| オプトアウト管理 | 配信停止リクエストの自動処理、配信停止リンク自動挿入 |
| セグメント | タグ・カスタムフィールドによるフィルタリング |

#### 2.1.2 メール作成
| 機能 | 詳細 |
|------|------|
| テンプレート | 再利用可能なメールテンプレートの作成・管理 |
| HTMLエディタ | リッチテキスト/HTMLでのメール作成 |
| プレースホルダー | `{{company_name}}`, `{{contact_name}}`, `{{custom_field}}` 等の差し込み |
| プレビュー | 送信前のプレビュー・テスト送信 |
| A/Bテスト | 件名・本文のバリエーション比較（将来拡張） |

#### 2.1.3 配信モード
| モード | 詳細 |
|--------|------|
| 即時配信 | リスト選択 → 送信実行 |
| 予約配信 | 日時指定で自動送信 |
| ステップメール | 登録日/前回送信からの経過日数に応じた自動シーケンス配信 |

#### 2.1.4 配信制御
| 機能 | 詳細 |
|------|------|
| 送信レート制限 | 時間あたりの送信数制限（レピュテーション保護） |
| バウンス処理 | ハードバウンス自動検出 → リストから除外 |
| 送信間隔 | メール間のランダム遅延（スパム回避） |
| ドメインローテーション | 複数送信ドメインの切り替え（将来拡張） |

#### 2.1.5 分析・レポート
| 指標 | 詳細 |
|------|------|
| 送信数/到達率 | 送信成功・失敗の集計 |
| 開封率 | トラッキングピクセルによる開封検知 |
| クリック率 | リンクトラッキング |
| 配信停止率 | オプトアウト率の推移 |
| バウンス率 | ハード/ソフトバウンスの内訳 |

### 2.2 フォーム営業機能

#### 2.2.1 フォーム検出
| 機能 | 詳細 |
|------|------|
| URL指定検出 | 企業URLを入力 → 問い合わせページ・フォームを自動検出 |
| 一括検出 | 企業リストビルダーのデータから複数企業のフォームを一括検出 |
| フォーム解析 | フォームのフィールド構成（name, email, company, message等）を自動解析 |
| フィールドマッピング | 検出されたフィールドとテンプレート変数の自動/手動マッピング |
| 検出結果保存 | 検出済みフォーム情報のキャッシュ（再利用） |

#### 2.2.2 フォーム送信
| 機能 | 詳細 |
|------|------|
| テンプレート送信 | 定型文テンプレートでの一括送信 |
| カスタマイズ送信 | 企業ごとにメッセージ内容を個別調整 |
| プレースホルダー | `{{company_name}}`, `{{industry}}` 等の企業情報差し込み |
| 送信スケジュール | 日時指定・分散送信（一度に大量送信しない） |
| CAPTCHA対応 | reCAPTCHA等がある場合はスキップ＆通知 |
| 送信前確認 | 送信内容のプレビュー・承認フロー |

#### 2.2.3 送信制御
| 機能 | 詳細 |
|------|------|
| 送信レート | 1企業あたりの送信間隔制御（最低24時間等） |
| 重複防止 | 同一企業への重複送信ブロック |
| ブラックリスト | 送信除外企業の管理 |
| リトライ | 一時的なエラー時の自動リトライ（上限付き） |

#### 2.2.4 分析・レポート
| 指標 | 詳細 |
|------|------|
| 送信成功率 | フォーム送信成功/失敗の集計 |
| 企業別ステータス | 検出成功/CAPTCHA/フォームなし/送信済みの分類 |
| 返信トラッキング | 返信メールの検知（受信メール連携、将来拡張） |

### 2.3 企業リストビルダー連携

| 機能 | 詳細 |
|------|------|
| データインポート | company-list-builderで検索・作成した企業リストを取り込み |
| API連携 | company-list-builderのAPIから企業データを直接取得 |
| フィールドマッピング | 企業名・業種・所在地・URL等を自動マッピング |
| 同期 | リスト更新時の差分同期 |

### 2.4 共通機能

#### 2.4.1 ダッシュボード
- メール配信の統計サマリー（送信数、開封率、クリック率）
- フォーム営業の統計サマリー（送信数、成功率）
- 直近の配信・送信スケジュール一覧
- アクションが必要な項目（バウンス、CAPTCHA検出等）のアラート

#### 2.4.2 キャンペーン管理
- メール配信とフォーム営業を「キャンペーン」単位で統合管理
- キャンペーンごとの進捗・成果トラッキング
- キャンペーンの複製・アーカイブ

#### 2.4.3 テンプレート管理
- メール用・フォーム営業用テンプレートの作成・編集・削除
- テンプレートカテゴリ分類
- テンプレートの共有（チーム内/公開テンプレート）

#### 2.4.4 ユーザー管理（SaaS向け）
| 機能 | 詳細 |
|------|------|
| 認証 | メール/パスワード、Google OAuth |
| プラン管理 | Free/Starter/Pro等の料金プラン |
| チーム | 複数ユーザーでの共有（将来拡張） |
| 利用制限 | プランに応じた月間送信数制限 |

## 3. 非機能要件

### 3.1 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js 14 (App Router), React, Tailwind CSS |
| バックエンド | Next.js API Routes + バックグラウンドジョブ |
| データベース | Supabase (PostgreSQL) |
| 認証 | Clerk |
| 決済 | Stripe |
| メール送信 | Resend (配信) + Amazon SES (大量配信時、将来) |
| フォーム検出/送信 | Playwright (ヘッドレスブラウザ) |
| ジョブキュー | Vercel Cron + Supabase Queue or BullMQ |
| ホスティング | Vercel |
| 監視 | Sentry |

### 3.2 パフォーマンス

| 項目 | 目標値 |
|------|--------|
| メール送信スループット | 1,000通/時間（初期） |
| フォーム送信スループット | 100件/時間（初期） |
| ダッシュボード応答 | 2秒以内 |
| CSVインポート | 10,000件/分 |

### 3.3 スケーラビリティ

| 項目 | 詳細 |
|------|------|
| 初期想定 | 月間10,000通メール + 1,000件フォーム送信 |
| 拡張目標 | 月間100,000通メール + 10,000件フォーム送信 |
| ジョブ実行 | Vercel Cron → 将来的にワーカー分離 |

### 3.4 セキュリティ

| 項目 | 対策 |
|------|------|
| 認証 | Clerk による認証・セッション管理 |
| API保護 | 認証必須 + レート制限 |
| データ暗号化 | Supabase TLS + RLS |
| メールコンプライアンス | 特定電子メール法準拠（オプトアウト機能必須） |
| ログ | 全送信ログの保存（監査対応） |

### 3.5 法令遵守

| 法令 | 対応 |
|------|------|
| 特定電子メール法 | 配信停止リンク必須、オプトイン/オプトアウト管理 |
| 個人情報保護法 | 個人情報の適切な取り扱い、利用目的の明示 |
| GDPR（将来） | EU向け配信時の同意管理 |

## 4. データモデル（概要）

### 4.1 主要テーブル

```
contacts          - コンタクト情報 (email, name, company, custom_fields)
contact_lists     - コンタクトリスト (name, description)
contact_list_members - リストとコンタクトの中間テーブル
email_templates   - メールテンプレート (subject, body_html, placeholders)
email_campaigns   - メールキャンペーン (list_id, template_id, schedule, status)
email_sends       - 個別送信ログ (campaign_id, contact_id, status, opened_at, clicked_at)
form_templates    - フォーム営業テンプレート (message, placeholders)
form_targets      - フォーム送信先 (company_name, url, form_url, field_mapping)
form_campaigns    - フォーム営業キャンペーン (target_list, template_id, schedule)
form_sends        - フォーム送信ログ (campaign_id, target_id, status, response)
step_sequences    - ステップメール定義 (steps, delays)
step_enrollments  - ステップメール登録状況 (sequence_id, contact_id, current_step)
users             - ユーザー (auth, plan, usage)
```

### 4.2 ER図（概念）

```
contacts ──< contact_list_members >── contact_lists
    │
    ├──< email_sends >── email_campaigns ── email_templates
    │
    └──< step_enrollments >── step_sequences

form_targets ──< form_sends >── form_campaigns ── form_templates

users ──< contact_lists
users ──< email_campaigns
users ──< form_campaigns
```

## 6. API設計（概要）

### 6.1 メール配信API

```
POST   /api/contacts/import          CSVインポート
GET    /api/contacts                  コンタクト一覧
POST   /api/contacts                  コンタクト追加
DELETE /api/contacts/:id              コンタクト削除
GET    /api/lists                     リスト一覧
POST   /api/lists                     リスト作成
POST   /api/campaigns/email           メールキャンペーン作成
POST   /api/campaigns/email/:id/send  配信実行
GET    /api/campaigns/email/:id/stats 配信統計
POST   /api/sequences                 ステップメール作成
```

### 6.2 フォーム営業API

```
POST   /api/form-finder/detect        フォーム検出
GET    /api/form-targets               送信先一覧
POST   /api/campaigns/form             フォームキャンペーン作成
POST   /api/campaigns/form/:id/send    送信実行
GET    /api/campaigns/form/:id/stats   送信統計
```

### 6.3 共通API

```
GET    /api/dashboard/stats            ダッシュボード統計
GET    /api/templates/email            メールテンプレート一覧
POST   /api/templates/email            テンプレート作成
GET    /api/templates/form             フォームテンプレート一覧
POST   /api/templates/form             テンプレート作成
POST   /api/webhooks/resend            Resend Webhook (バウンス等)
```

## 8. 開発フェーズ

### Phase 1: MVP（メール配信基盤）
- CSVインポート
- コンタクトリスト管理
- テンプレート作成
- 即時/予約配信
- 送信ログ・基本統計
- オプトアウト管理

### Phase 2: フォーム営業
- フォーム自動検出（Playwright）
- フィールドマッピング
- テンプレート送信
- 送信ログ・統計
- CAPTCHA検出・スキップ

### Phase 3: ステップメール
- シーケンス定義
- 自動エンロール
- 条件分岐（開封/クリック時）
- シーケンス分析

### Phase 4: SaaS化
- 認証・課金（Clerk + Stripe）
- プラン別制限
- ユーザーダッシュボード
- ランディングページ

### Phase 5: 高度な機能
- 開封/クリックトラッキング
- A/Bテスト
- 企業リストビルダー連携API
- チーム機能
- ドメインローテーション

## 9. リスク・考慮事項

| リスク | 対策 |
|--------|------|
| メール到達率低下 | SPF/DKIM/DMARC設定、送信レピュテーション管理、ウォームアップ |
| スパム判定 | 送信レート制御、コンテンツチェック、オプトアウト遵守 |
| フォーム送信ブロック | CAPTCHA検出時スキップ、IP分散（将来）、User-Agent管理 |
| 法的リスク | 特定電子メール法遵守、利用規約整備、不正利用防止 |
| Vercel制約 | 実行時間制限（Hobbyプラン10秒）→ 長時間ジョブはCron分割 or 外部ワーカー |
| Resend制限 | 月間送信上限 → SES併用検討 |
