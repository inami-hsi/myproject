# エバーグリーンローンチサイト 要件定義書

**プロジェクト名**: Evergreen Launch Platform
**Phase**: 1 - Requirements
**作成日**: 2026-03-04
**ステータス**: Draft

---

## 1. プロジェクト概要

### 1.1 目的

録画済み動画を活用したエバーグリーンローンチ（自動化ウェビナーファネル）を実現するWebプラットフォーム。
ユーザーはランディングページから日程を選択して登録し、指定時間に動画を視聴、視聴前後に自動メールが配信される仕組みを構築する。

### 1.2 ビジネスゴール

- 24時間365日、自動で商品・サービスのローンチを実施
- 見込み客のメールアドレスを取得し、ナーチャリング（育成）を自動化
- 動画視聴後のオファー提示で売上を最大化
- 運営コストを最小限に抑えた持続可能なマーケティングシステム

### 1.3 ターゲットユーザー

| ユーザー種別 | 説明 |
|-------------|------|
| **視聴者（エンドユーザー）** | ランディングページから登録し動画を視聴する見込み客 |
| **管理者（サイトオーナー）** | キャンペーン・動画・メールを管理する運営者 |

## 2. 機能要件

### FR-001: ランディングページ（LP）

- **優先度**: P0-Critical
- **説明**: 見込み客が日程を選択し、メールアドレスを登録できるページ
- **受入基準**:
  - [ ] キャンペーンごとにLPを表示できる
  - [ ] 複数の日程（セッション）が選択肢として表示される
  - [ ] 日程はエバーグリーン方式で自動生成される（例: 今日から3日後、5日後、7日後）
  - [ ] 名前・メールアドレスの入力フォームがある
  - [ ] 登録後に確認ページ（サンキューページ）へリダイレクトされる
  - [ ] LP内にカウントダウンタイマーを表示できる
  - [ ] モバイルレスポンシブ対応

### FR-002: セッション日程管理

- **優先度**: P0-Critical
- **説明**: エバーグリーン方式でセッション日程を自動生成・管理する
- **受入基準**:
  - [ ] 管理者がセッションのルール（何日後、何時開始）を設定できる
  - [ ] 訪問者のアクセス時刻に基づいて「次回のセッション日程」が動的に生成される
  - [ ] 過去のセッション日程は自動的に非表示になる
  - [ ] タイムゾーン対応（訪問者のローカル時間で表示）
  - [ ] セッション開始前のカウントダウン表示

### FR-003: 動画管理（管理者）

- **優先度**: P0-Critical
- **説明**: 管理者がローンチ用動画をアップロード・管理できる
- **受入基準**:
  - [ ] 動画ファイルをアップロードできる
  - [ ] 動画メタデータ（タイトル、説明、サムネイル）を設定できる
  - [ ] 動画をキャンペーンに紐付けできる
  - [ ] 動画の公開/非公開を切り替えできる
  - [ ] アップロード済み動画の一覧を確認できる

### FR-004: 動画視聴ページ

- **優先度**: P0-Critical
- **説明**: 登録者が指定時間に動画を視聴できるページ
- **受入基準**:
  - [ ] セッション開始前はカウントダウンタイマーを表示
  - [ ] セッション開始時刻になると動画プレーヤーが表示される
  - [ ] 動画はストリーミング再生される
  - [ ] 動画の不正ダウンロード防止（右クリック無効、URL難読化）
  - [ ] 視聴中の離脱検知（視聴完了率のトラッキング）
  - [ ] 動画終了後にCTA（Call to Action）ボタンを表示
  - [ ] レスポンシブ対応（モバイルでも快適に視聴可能）

### FR-005: メール自動配信

- **優先度**: P0-Critical
- **説明**: 登録者に対して視聴前後のメールを自動送信する
- **受入基準**:
  - [ ] 登録直後の確認メール送信
  - [ ] セッション24時間前のリマインダーメール
  - [ ] セッション1時間前のリマインダーメール
  - [ ] セッション開始時の「今すぐ視聴」メール
  - [ ] 視聴後のフォローアップメール（視聴完了者向け）
  - [ ] 未視聴者向けリプレイ案内メール
  - [ ] メール内容をキャンペーンごとにカスタマイズ可能
  - [ ] 管理者がメールテンプレートを編集できる
  - [ ] 送信タイミングをカスタマイズ可能
  - [ ] メール送信ログの確認

### FR-006: 決済・オファー

- **優先度**: P1-High
- **説明**: 動画視聴後に商品・サービスの購入フローを提供する
- **受入基準**:
  - [ ] 動画終了後にオファーページ（セールスページ）を表示
  - [ ] Stripeを使った決済（一回払い・分割払い）
  - [ ] サブスクリプション（月額課金）にも対応
  - [ ] 決済完了後の確認メール自動送信
  - [ ] 期間限定オファー（カウントダウンタイマー付き）
  - [ ] クーポンコード対応

### FR-007: 管理者ダッシュボード

- **優先度**: P1-High
- **説明**: キャンペーン全体を管理・分析するダッシュボード
- **受入基準**:
  - [ ] キャンペーン作成・編集・削除
  - [ ] 登録者一覧の確認
  - [ ] メール送信状況の確認
  - [ ] 視聴率・コンバージョン率の確認
  - [ ] 売上レポート
  - [ ] メールテンプレート管理

### FR-008: ユーザー認証

- **優先度**: P1-High
- **説明**: 管理者のログイン認証
- **受入基準**:
  - [ ] Clerk を使用した管理者ログイン
  - [ ] メール + パスワード認証
  - [ ] Google OAuth ログイン対応
  - [ ] 管理者権限の管理

### FR-009: キャンペーン管理

- **優先度**: P1-High
- **説明**: 複数のエバーグリーンキャンペーンを運用できる
- **受入基準**:
  - [ ] キャンペーンの作成（名前、説明、動画、メール設定）
  - [ ] キャンペーンごとに専用LP URLが生成される
  - [ ] キャンペーンの有効/無効切り替え
  - [ ] キャンペーンの複製（テンプレート化）

### FR-010: 分析・トラッキング

- **優先度**: P2-Medium
- **説明**: マーケティング指標の計測・分析
- **受入基準**:
  - [ ] LP訪問数（PV）
  - [ ] 登録率（CVR）
  - [ ] メール開封率・クリック率
  - [ ] 動画視聴完了率
  - [ ] 購入コンバージョン率
  - [ ] PostHog連携によるイベントトラッキング
  - [ ] UTMパラメータ対応

## 4. 技術制約

### TC-001: 技術スタック

> **方針**: 無料プランを最大限活用し、初期コストを最小化する

| カテゴリ | 技術 | 無料枠 | 用途 |
|----------|------|--------|------|
| **フレームワーク** | Next.js 15 (App Router) | - | フロントエンド + API Routes |
| **言語** | TypeScript 5.3+ | - | strict mode |
| **バックエンド/DB** | Supabase | 500MB DB, 1GB Storage, 50K MAU | DB・ファイルストレージ・Realtime |
| **デプロイ** | Vercel | 100GB帯域/月 | ホスティング・CDN・Edge Functions |
| **DNS/CDN** | Cloudflare | 無制限 | DNS管理・WAF・キャッシュ |
| **認証** | Clerk | 10,000 MAU | 管理者認証 |
| **決済** | Stripe | 売上の3.6% | クレジットカード決済 |
| **メール** | Resend | 3,000通/月 | トランザクションメール |
| **キャッシュ** | Upstash Redis | 10,000 req/日 | レート制限・セッション管理 |
| **分析** | PostHog | 100万イベント/月 | プロダクト分析 |
| **エラー監視** | Sentry | 5,000イベント/月 | エラートラッキング |
| **バージョン管理** | GitHub | 無制限 | ソースコード管理 |
| **ドメイン** | Namecheap | ¥1,800/年 | 独自ドメイン |

### TC-002: 動画ストレージ戦略

Supabase Storageの無料枠（1GB）では動画保存に不足する可能性があるため、以下の戦略を採用:

| 方法 | コスト | メリット | デメリット |
|------|--------|---------|-----------|
| **Cloudflare R2** | 10GB無料、エグレス無料 | 大容量・高速配信 | 設定がやや複雑 |
| **YouTube限定公開 + 埋め込み** | 無料・無制限 | 容量無制限・トランスコード不要 | YouTube UIが出る・制御が限定的 |
| **Supabase Storage** | 1GB無料 | 統合管理しやすい | 容量制限あり |

**推奨**: Cloudflare R2 をメイン動画ストレージとし、HLSストリーミングで配信。
短い動画や少数キャンペーンの場合は Supabase Storage で十分。

### TC-003: メール送信制限への対策

Resend無料枠は月3,000通。以下の運用ルールで対応:

| 指標 | 計算 |
|------|------|
| 1登録者あたり最大メール数 | 6通（確認+リマインダー3通+開始+フォロー） |
| 月間最大登録者数 | 約500人（3,000 ÷ 6） |
| 超過時の対応 | Resend有料プラン（$20/月で50,000通）へアップグレード |

### TC-004: 使用しない技術

以下はこのプロジェクトでは使用しない（CCAGI SDK標準スタックとの差異）:

| 技術 | 理由 |
|------|------|
| AWS (ECS/ECR/RDS) | Vercel + Supabase で代替（無料枠活用） |
| Prisma | Supabase Client で直接アクセス |
| Docker | Vercel のサーバーレスデプロイを使用 |
| @customer-cloud/platform-sdk | CC-Auth不使用（Clerk使用） |
| Pinecone | AI機能は現時点で不要（将来拡張可能） |

## 8. メールシーケンス設計

### 標準メールシーケンス

```
登録完了
  │
  ├── [即時] 登録確認メール
  │     └─ セッション日時、視聴URL、準備事項
  │
  ├── [24時間前] リマインダー①
  │     └─ 明日のセッション案内、期待感醸成
  │
  ├── [1時間前] リマインダー②
  │     └─ まもなく開始、直接リンク
  │
  ├── [開始時] 開始通知
  │     └─ 「今すぐ視聴」ボタン
  │
  ├── [終了後1時間] フォローアップ（視聴者向け）
  │     └─ オファー案内、限定特典
  │
  └── [終了後24時間] リプレイ案内（未視聴者向け）
        └─ リプレイ視聴リンク、期限付き
```

### メール送信タイミング

| トリガー | delay_minutes | 対象 |
|---------|--------------|------|
| confirmation | 0（即時） | 全登録者 |
| reminder_24h | -1440 | 全登録者 |
| reminder_1h | -60 | 全登録者 |
| start | 0（セッション開始時） | 全登録者 |
| followup | +60 | 視聴完了者 |
| replay | +1440 | 未視聴者 |

## 10. バックグラウンドジョブ設計

エバーグリーンの自動化にはスケジュールジョブが必要:

| ジョブ | 実行間隔 | 処理内容 |
|--------|---------|---------|
| メール送信チェック | 5分ごと | 送信タイミングに達したメールを送信 |
| セッション生成 | 1時間ごと | 未来のセッション日程を自動生成 |
| 期限切れセッションクリーンアップ | 1日1回 | 古いセッションデータの削除 |

**実装方法**: Vercel Cron Jobs（vercel.json で設定）+ Supabase Edge Functions

```json
{
  "crons": [
    { "path": "/api/cron/send-emails", "schedule": "*/5 * * * *" },
    { "path": "/api/cron/generate-sessions", "schedule": "0 * * * *" },
    { "path": "/api/cron/cleanup", "schedule": "0 3 * * *" }
  ]
}
```

**注意**: Vercel無料プランでは Cron Jobs は1日1回まで。
5分間隔のメール送信チェックには Vercel Pro（$20/月）が必要、
または Supabase の pg_cron + Edge Functions で代替。
