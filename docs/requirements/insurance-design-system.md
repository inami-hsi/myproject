# 保険商品推奨システム - デザインシステム＆ワイヤーフレーム詳細設計

**プロジェクト名**: 保険商品推奨ポータル  
**作成日**: 2026年2月27日  
**版**: v1.0

---

## 1. デザイン方針

### 1.1 プロジェクト全体のデザイン哲学

```
「複雑な保険選びをシンプルに」
「信頼感と親しみやすさの両立」
「段階的な訴求 → わかりやすい結果」
```

### 1.2 ユーザーペルソナ別の期待値

| ペルソナ | 年代 | デバイス | 優先度 |
|---------|------|--------|--------|
| **P1**: オンライン自動加入派 | 25-40 | スマートフォン | 「速さ」「シンプルさ」 |
| **P2**: 対面相談志向派 | 40-60 | PC＋タブレット | 「詳細さ」「信頼感」 |
| **P3**: 比較検討派 | 30-50 | PC | 「比較表」「説得力」 |

### 1.3 デザインシステムの基本原則

✓ **モバイルファースト** → PC対応  
✓ **段階的フロー** → 1ページ1アクションで集中力維立  
✓ **視覚的優先度** → 推奨1位は目立つ、2-3位は淡い  
✓ **根拠の透明性** → 「なぜこの会社か」を必ず記載  
✓ **心理的安心** → 保険業法準拠、ロ方式表明

## 7. インタラクション・アニメーション仕様

### 7.1 ボタンホバー・クリック時

```
Primary Button ホバー時:
├─ 背景色: #0066CC → #0052A3 (即座に)
├─ シャドウ: なし → 0 2px 8px rgba(0,0,0,0.1)
└─ トランジション: 200ms ease-out

クリック時（フィードバック）:
├─ スケール: 100% → 98% (わずかに縮小)
├─ 続いて: 98% → 100% に戻る
└─ トランジション: 100ms

Disabled状態:
├─ Opacity: 0.6
└─ Cursor: not-allowed
```

### 7.2 ページ遷移アニメーション

```
質問ページ → 次の質問ページ
├─ 前: フェードアウト (150ms)
├─ 後: フェードイン (150ms)
└─ 効果: スムーズな流動感を演出

結果ページ表示時:
├─ 1位推奨: スライドダウン (300ms)
├─ 2位: フェードイン（遅延 100ms）
├─ 3位: フェードイン（遅延 200ms）
└─ 目的: 段階的な視覚的導線
```

### 7.3 詳細説明の展開・畳み

```
トグルボタンクリック時:
├─ 詳細セクション: 高さ 0 → auto (300ms)
├─ テキスト: グレー → ブラック（色遷移）
├─ 矢印: ▼ ↔ ▲ (180度回転)
└─ 背景: transparent → #E1F0FF (軽く)
```
