# 保険商品推奨システム - 要件定義書

**プロジェクト名**: 保険商品推奨ポータルサイト構築
**作成日**: 2026年2月27日
**版**: v1.0

## 1. プロジェクト概要

顧客が自身のニーズを段階的に選択することで、最適な保険商品と推奨保険会社を提示するWebベースのガイダンスシステム。

保険業法に基づくロ方式（顧客の意向に基づくプロセス）を採用し、一般的な保険選定ポイントを軸としたヒアリングを通じて、合理的な推奨を実現。

## 2. 対象保険商品

### 2.1 損害保険（個人向け）

- **自動車保険**
  - 運転頻度（毎日/週数回/月1回程度）
  - 年間走行距離（5,000km以下/5,000～10,000km/10,000km以上）
  - 過去の事故歴（無/あり）
  - 運転者管理（本人のみ/配偶者/子ども含む）
  
- **火災保険**
  - 対象物（戸建/マンション/賃貸）
  - 主な補償対象（建物のみ/家財のみ/両方）
  - 災害リスク（豪雨/雹/雪害の重要度）
  - 特約ニーズ（盗難/破損/水漏れ）

- **賠償責任保険**
  - 補償対象（日常生活/趣味・スポーツ/ペット飼育）
  - 居住エリアのリスク意識

- **傷害保険（労災上乗せ）**
  - 職業分類（一般職/現場作業/危険作業）
  - 就業形態（正社員/自営業/パート）

### 2.2 損害保険（法人向け）

- 自動車保険（事業用）
- 火災保険（事業所）
- 賠償責任保険（事業活動）
- 傷害保険

### 2.3 生命保険（個人向け）

- **死亡保険**
  - 定期保険
    - 必要保障額
    - 保障期間（子どもの成人まで/定年まで）
  - 終身保険
    - 相続対策/遺産分割対策への関心度
  - 収入保障保険
    - 扶養家族の生活費カバー重視
  - 就業不能保険
    - 生涯所得補保対策

- **医療保険**
  - 入院給付日額（5,000円/10,000円/20,000円以上）
  - 通院補償の必要性
  - 手術特約への関心度

- **がん保険**
  - 診断給付金の優先度
  - 先進医療特約への関心度
  - 一時金vs継続給付の選好

- **個人年金保険**
  - 貯蓄型or返戻率重視
  - 運用リスク許容度（保証型/変額）

- **変額保険**
  - 資産形成志向
  - 運用リスク許容度（積極/中庸/保守）

- **一時払い終身保険**
  - 相続対策
  - 資産活用効率

- **引き受け基準緩和型保険**
  - 既往症/持病あり
  - 入院歴

### 2.4 生命保険（法人向け）

- 経営者向け死亡保険
- 従業員福利厚生医療保険
- 団体所得補保保険

## 4. ロ方式ヒアリング設計

### 4.1 ロ方式とは

保険業法に基づく顧客適合性確認プロセス。顧客の**意向・ニーズ・知識・経験**を把握した上で、合理的な保険商品を推奨するアプローチ。

ハ方式（一律的な基準）ではなく、個別の判断を重視。

### 4.2 ヒアリング項目のレイヤー構成

**レイヤー1: 保険の種類選択（入り口）**
- 損害保険 vs 生命保険

**レイヤー2: 具体的な保険種目選択**
- 損保の場合: 自動車/火災/賠償/傷害
- 生保の場合: 死亡/医療/がん/年金等

**レイヤー3: ニーズヒアリング（コア）**

各保険種目ごとに、以下の軸で質問を設計：

1. **ライフステージ/人口統計**
   - 年齢層
   - 家族構成
   - 職業分類

2. **保障ニーズの優先度**
   - 「貯蓄性」vs「保障性」のバランス
   - 「保険料安さ」vs「特約充実」のバランス

3. **リスク認識**
   - 過去の事故経験
   - リスク意識の高さ

4. **サービス期待**
   - 対面相談重視
   - オンライン・デジタル重視
   - カスタマーサービス充実度

5. **付加価値**
   - 健康診断・予防サービス
   - アフターサービス

### 4.3 各種目別ヒアリング項目例

#### 自動車保険

```
Q1. 運転頻度は？
 → 毎日
 → 週数回
 → 月1回程度

Q2. 年間走行距離は？
 → 5,000km以下
 → 5,000～10,000km
 → 10,000km以上

Q3. 過去3年の事故歴は？
 → なし
 → 物損事故あり
 → 人身事故あり

Q4. 運転者の範囲は？
 → 本人のみ
 → 配偶者まで
 → 家族全体

Q5. 優先する補償は？
 → 保険料の安さ
 → 事故対応の手厚さ
 → 特約の充実
```

#### 火災保険

```
Q1. 対象物は？
 → 戸建て（持家）
 → マンション（持家）
 → 賃貸住宅

Q2. 主な保障対象は？
 → 建物のみ
 → 家財のみ
 → 建物＋家財

Q3. 災害リスク認識は？
 → 豪雨・平成30年西日本豪雨
 → 雹による屋根被害
 → 雪害

Q4. 特約への関心度は？
 → 基本補償で十分
 → 水漏れ・盗難も必要
 → フル補償重視

Q5. 保険料体感は？
 → 安さ最優先
 → バランス重視
 → サービス充実重視
```

#### 医療保険

```
Q1. ライフステージは？
 → 若年層（20-30代）
 → 中年層（40-50代）
 → シニア層（60代以上）

Q2. 家計のゆとりは？
 → 月額3,000円程度
 → 月額5,000-7,000円
 → 月額10,000円以上

Q3. 入院給付日額ニーズは？
 → 5,000円/日
 → 10,000円/日
 → 20,000円/日以上

Q4. 通院補償への関心？
 → 不要
 → あると良い
 → 重視

Q5. サービス面での優先？
 → 保険料安さ
 → 診断給付金の手厚さ
 → 先進医療対応
 → 健康サービス充実
```

#### がん保険

```
Q1. がんへの関心背景は？
 → 家族歴がある
 → 健診で指摘された経験
 → 一般的なリスク対策

Q2. 優先する給付形態？
 → 診断給付金（一括）
 → 継続給付金（月額）
 → 両方

Q3. 先進医療への関心？
 → 関心なし
 → あると良い
 → 必須

Q4. 既往症や既検査は？
 → 健康診断OK
 → 健診所見あり
 → 既往症あり
```

## 5. 推奨ロジック設計

### 5.1 推奨決定木の概要

```
顧客入力
  ↓
保険種目選択（損保/生保）
  ↓
具体的な種目選択
  ↓
ヒアリング質問に回答（3～5問）
  ↓
スコアリング計算
  ↓
保険会社X商品の推奨理由を生成
  ↓
複数案（1位/2位/3位）表示
  ↓
推奨理由を詳細表示（簡潔/詳細の選択）
```

### 5.2 推奨スコアリング要素

各保険会社には、以下の軸でスコアを定義：

| スコア軸 | 内容 | 加点条件 |
|---------|------|--------|
| **保険料競争力** | 平均的な保険料水準 | Q「保険料重視」で加点 |
| **事故対応評価** | 顧客満足度・対応速度 | Q「事故対応重視」で加点 |
| **特約充実度** | 提供される特約の種類・柔軟性 | Q「カスタマイズ重視」で加点 |
| **デジタル対応** | オンライン手続き・申込の容易性 | Q「デジタル重視」で加点 |
| **サービス品質** | HDI格付等の外部評価 | Q「サービス重視」で加点 |
| **商品特異性** | その会社にしかない商品・特徴 | 該当商品で加点 |
| **持病対応** | 引き受け基準の緩さ | Q「既往症あり」で加点 |

### 5.3 推奨説明パターン

#### 簡潔説明（1-2行）
```
「当社は事故対応の迅速さで業界評価が高く、あなたが『対応品質重視』を選んだため、1位推奨です。」
```

#### 詳細説明（3-5段落）
```
【推奨理由：詳細版】

1. あなたの選択とマッチ度
   - あなたが重視：「事故対応の手厚さ」「サービス品質」
   - 当社の強み：顧客満足度92% (HDI-Japan調査)、全国損害サービス拠点200か所

2. 競合他社との比較
   - 他社の対応拠点：平均150か所
   - 他社の顧客満足度：平均85%

3. 具体的なサービス内容
   - 24時間365日事故受付
   - 弁護士約550名、顧問医約100名の配置
   - 修理工場の直接手配

4. 保険料の目安
   - あなたの条件（年間走行距離10,000km、事故歴なし）
   - 月額保険料：約4,500円（相場：4,300-5,200円）
```

## 6. UI/UXフロー概要

### 6.1 ページ構成

```
1. ランディングページ
   ├─ 損害保険ボタン → 損保フロー開始
   └─ 生命保険ボタン → 生保フロー開始

2. 保険種目選択ページ
   ├─ [損保の場合] 自動車/火災/賠償/傷害から選択
   └─ [生保の場合] 死亡/医療/がん/年金等から選択

3. ヒアリングフロー（3～5ページ）
   ├─ 質問1（ライフステージ等）
   ├─ 質問2（ニーズプライオリティ）
   ├─ 質問3（当社評価軸）
   ├─ 質問4（サービス期待）
   └─ 質問5（追加ニーズ）

4. 推奨結果ページ
   ├─ 1位推奨保険会社＋商品
   ├─ 推奨理由（簡潔/詳細切り替え）
   ├─ 2位・3位候補の表示
   ├─ 各社の詳細情報へのリンク
   ├─ 保険料見積例（目安）
   └─ 「相談する」「申し込む」ボタン

5. 比較表ページ（オプション）
   └─ 推奨3社の商品・特徴を並べた表
```

### 6.2 デザイン方針

- **レスポンシブ**: スマートフォン優先
- **段階的フロー**: 1質問1ページで集中力維持
- **視覚的優先度**: 推奨1位（大きく・色付け）> 2位・3位（小さく）
- **推奨根拠の透明性**: 理由が必ず記載される
- **早期終了可能**: 「これ以上回答したくない」場合は推奨結果を表示
