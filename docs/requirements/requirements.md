## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | TaskFlow（仮称） |
| 種別 | 個人向けタスク管理アプリ（ガントチャート付き） |
| 想定利用者 | ~10名 |
| 目的 | タスクの可視化とスケジュール管理の効率化 |
| リポジトリ | my-project |

### 1.1 プロジェクトビジョン

タスク管理とガントチャートを統合し、個人およびスモールチームのスケジュール管理を直感的に行えるWebアプリケーション。従来のガントチャートツールが持つ「密度の高さ＝使いにくさ」を排除し、穏やかなUIでタスクの流れを俯瞰できることを目指す。

### 1.2 スコープ

- **In Scope**: タスクCRUD、ガントチャート、カンバンボード、カレンダービュー、認証、リアルタイム同期
- **Out of Scope**: チャット機能、ファイル添付、ワークフロー自動化、外部サービス連携（Slack, Jira等）

## 2. 機能要件

### 2.1 コア機能

#### F-001: タスクCRUD

タスクの作成・表示・更新・削除を行う基本機能。

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| title | string | Yes | タスク名（最大200文字） |
| description | string | No | 説明（Markdown対応） |
| startDate | date | No | 開始日 |
| dueDate | date | No | 終了日 |
| priority | enum | Yes | Critical / High / Medium / Low |
| status | enum | Yes | 未着手 / 進行中 / 完了 / 保留 |
| progress | number | Yes | 0-100% |
| tags | string[] | No | タグ（カラーラベル） |
| projectId | string | Yes | 所属プロジェクト |
| assignee | string | No | 担当者 |

#### F-002: ガントチャート表示

プロジェクト全体のタイムラインをガントチャートで可視化する。

- 日 / 週 / 月 ビュー切替
- タイムライン上にタスクバーを表示
- タスク依存関係を矢印で表示
- 今日線（赤い点線）を表示
- ガントバー内に進捗率をインライン表示（幅が十分な場合）
- ズーム操作（ピンチ / スクロール）
- 描画パフォーマンス: 100タスクで500ms以内

#### F-003: 期限管理

- 開始日・終了日の設定（カレンダーピッカー）
- 期限超過タスクのハイライト表示（赤色）
- 期限前通知（ブラウザ通知、1日前・3日前・7日前）
- 期限なしタスクのバックログ表示

#### F-004: 進捗管理

- 0-100% スライダーによる進捗率更新
- ステータス自動連動: 0% -> 未着手、1-99% -> 進行中、100% -> 完了
- ガントバーの塗りつぶしで進捗を視覚表示
- プロジェクト全体の進捗率自動集計

#### F-005: タスク依存関係

- 依存関係タイプ: FS（Finish-to-Start）/ SS / FF / SF の4種
- ガントチャート上に矢印で依存関係を表示
- 循環依存の自動検出とエラー表示
- クリティカルパスのハイライト表示

#### F-006: マイルストーン

- ガントチャート上にダイヤモンド形状で表示
- マイルストーン名と日付のラベル表示
- マイルストーンの作成・編集・削除

#### F-007: プロジェクト管理

- 複数プロジェクトの作成・切替
- プロジェクトアーカイブ機能
- プロジェクト別のダッシュボード（進捗率、タスク数、期限超過数）
- 最大50プロジェクト/ユーザー

### 2.2 補助機能

#### F-008: フィルタリング

- ステータス別フィルタ
- 期限範囲フィルタ

## 5. ユーザーストーリー

### US-001: タスク作成

> ユーザーとして、タスクを作成し期限を設定できること

**受入基準:**
- [ ] タイトルを入力してタスクを作成できる
- [ ] 開始日・終了日をカレンダーピッカーで設定できる
- [ ] 優先度を4段階から選択できる
- [ ] 作成したタスクがリストとガントチャートに即座に反映される

### US-002: ガントチャート俯瞰

> ユーザーとして、ガントチャートでプロジェクト全体のスケジュールを俯瞰できること

**受入基準:**
- [ ] プロジェクトの全タスクがタイムライン上にバーとして表示される
- [ ] 日 / 週 / 月 のビュー切替ができる
- [ ] 今日線が赤い点線で表示される
- [ ] 100タスクでも500ms以内に描画される

### US-003: 依存関係とクリティカルパス

> ユーザーとして、タスクの依存関係を設定しクリティカルパスを把握できること

**受入基準:**
- [ ] タスク間にFS/SS/FF/SF の依存関係を設定できる
- [ ] 依存関係が矢印で表示される
- [ ] 循環依存を設定しようとするとエラーが表示される
- [ ] クリティカルパスがハイライト表示される

### US-004: ドラッグ&ドロップ

> ユーザーとして、ガントチャート上でドラッグ&ドロップでタスクを移動できること

**受入基準:**
- [ ] ガントバーをドラッグしてタスクの開始日・終了日を変更できる

- [ ] ドラッグ中にゴーストプレビューが表示される
- [ ] 依存関係のあるタスクは警告が表示される

### US-005: 進捗率更新

> ユーザーとして、タスクの進捗率を更新しリアルタイムにガントバーに反映されること

**受入基準:**
- [ ] スライダーで0-100%の進捗率を設定できる
- [ ] ガントバーの塗りつぶしが進捗率に連動する
- [ ] 100%設定時にステータスが自動で「完了」に変わる
- [ ] プロジェクト全体の進捗率が自動集計される

### US-006: カンバンボード

> ユーザーとして、カンバンボードでステータス別にタスクを管理できること

**受入基準:**
- [ ] 未着手 / 進行中 / 完了 / 保留 のカラムが表示される
- [ ] タスクカードをドラッグしてカラム間を移動できる
- [ ] カラム間移動でステータスが自動更新される
- [ ] 各カラムのタスク数が表示される

### US-007: 期限通知

> ユーザーとして、期限が近いタスクの通知を受け取れること

**受入基準:**
- [ ] 期限1日前にブラウザ通知を受け取れる

- [ ] アプリ内通知ベルに未読通知数が表示される

### A. 用語集

| 用語 | 説明 |
|------|------|
| FS (Finish-to-Start) | 先行タスクの完了後に後続タスクを開始 |
| SS (Start-to-Start) | 先行タスクの開始と同時に後続タスクを開始 |
| FF (Finish-to-Finish) | 先行タスクの完了と同時に後続タスクを完了 |
