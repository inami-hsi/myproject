**Phase**: 1 - Requirements
**作成日**: 2026-03-04
**ステータス**: Draft

## 1. プロジェクト概要

### 1.1 目的

日本全国の法人情報を**業種**と**地域（都道府県・市区町村）**で絞り込み、営業リスト・マーケティングリストを即座に作成・ダウンロードできるWebアプリケーション。

政府公開データ（gBizINFO・国税庁法人番号）を活用し、**データ取得コストゼロ**で競合より大幅に安価なサービスを提供する。

### 1.2 データソース

| データソース | 提供元 | 費用 | 件数 | 取得可能情報 |
|-------------|--------|------|------|-------------|
| **gBizINFO REST API** | 経済産業省 | **無料** | 約500万法人 | 法人番号、法人名、所在地、代表者名、資本金、従業員数、営業品目（業種）、事業概要、企業HP、設立年 |
| **国税庁法人番号 Web-API** | 国税庁 | **無料** | 約600万法人 | 法人番号、商号、所在地（基本3情報） |
| **国税庁全件データ** | 国税庁 | **無料** | 全法人 | CSV一括ダウンロード（都道府県別、月次更新） |

> **戦略**: gBizINFOをメインデータソース（業種・詳細情報あり）、国税庁データを補完データソース（最新の法人名・所在地）として使用。両方を法人番号で名寄せ・統合する。

### 1.3 ビジネスモデル

| 項目 | 内容 |
|------|------|
| 収益モデル | フリーミアム + 月額サブスクリプション |
| ターゲット価格 | ¥0（Free）〜 ¥9,800/月（Pro） |
| 無料プラン | あり（月50件ダウンロード） |
| 差別化 | データ取得コスト0 → 競合比80%安 |

### 1.4 ターゲットユーザー

| ユーザー種別 | 説明 | 利用シーン |
|-------------|------|-----------|
| **BtoB営業担当** | 新規開拓のためのターゲットリスト作成 | 「東京都の飲食業」「大阪府の製造業」で絞込 |
| **マーケティング担当** | DM・メール配信対象のリスト作成 | 業種×地域×従業員数で精密ターゲティング |
| **中小企業診断士** | 地域の企業調査・分析 | 特定市区町村の業種別企業数把握 |
| **スタートアップ** | 市場調査・パートナー企業探索 | 特定業種の企業一覧+HP情報取得 |
| **自治体・商工会議所** | 地域企業の把握・支援策立案 | 管轄地域の法人数・業種分布の可視化 |

## 2. 機能要件

### FR-001: 業種検索

- **優先度**: P0-Critical
- **説明**: 日本標準産業分類に基づく業種で企業を絞り込む
- **受入基準**:
  - [ ] 日本標準産業分類（大分類〜小分類）の階層ツリーから業種を選択できる
  - [ ] 大分類（20種: A農業〜T分類不能）での絞り込み
  - [ ] 中分類（99種）での絞り込み
  - [ ] 小分類での絞り込み
  - [ ] 複数業種の同時選択（OR条件）
  - [ ] 業種名のテキスト検索（オートコンプリート付き）
  - [ ] gBizINFOの営業品目データとのマッピング

### FR-002: 地域検索

- **優先度**: P0-Critical
- **説明**: 都道府県・市区町村で企業を絞り込む
- **受入基準**:
  - [ ] 47都道府県での絞り込み
  - [ ] 市区町村での絞り込み（都道府県選択後にカスケード表示）
  - [ ] 複数都道府県・市区町村の同時選択（OR条件）
  - [ ] 地方区分での一括選択（北海道、東北、関東、中部、近畿、中国、四国、九州・沖縄）
  - [ ] 地域名のテキスト検索（オートコンプリート付き）
  - [ ] 全国市区町村コード（総務省）との対応

### FR-003: 詳細フィルタ

- **優先度**: P1-High
- **説明**: 業種・地域以外の条件で企業を絞り込む
- **受入基準**:
  - [ ] 資本金の範囲指定（例: 1,000万〜1億円）
  - [ ] 従業員数の範囲指定（例: 10人〜100人）
  - [ ] 設立年の範囲指定
  - [ ] 法人種別（株式会社、合同会社、有限会社、一般社団法人等）
  - [ ] Webサイトの有無
  - [ ] キーワード検索（事業概要・法人名の部分一致）
  - [ ] 複数フィルタの AND 結合

### FR-004: 検索結果表示

- **優先度**: P0-Critical
- **説明**: フィルタ条件に合致する企業を一覧表示する
- **受入基準**:
  - [ ] 検索結果の件数をリアルタイム表示
  - [ ] 一覧テーブル表示（ソート・ページネーション対応）
  - [ ] 表示カラムの選択（法人名、所在地、業種、資本金、従業員数、代表者、HP等）
  - [ ] 企業詳細のモーダル表示（クリックで詳細情報）
  - [ ] 法人番号公表サイト・gBizINFOへの外部リンク
  - [ ] 検索結果のリアルタイムプレビュー（最初の20件）

### FR-005: リストダウンロード

- **優先度**: P0-Critical
- **説明**: 検索結果を営業リストとしてダウンロードする
- **受入基準**:
  - [ ] CSV形式でのダウンロード
  - [ ] Excel（.xlsx）形式でのダウンロード
  - [ ] ダウンロード対象カラムの選択
  - [ ] ダウンロード件数の表示（プラン制限の残数表示）
  - [ ] 文字コード選択（UTF-8 / Shift-JIS）
  - [ ] ダウンロード履歴の保存

### FR-006: 検索条件の保存・共有

- **優先度**: P1-High
- **説明**: よく使う検索条件を保存し、再利用・共有できる
- **受入基準**:
  - [ ] 検索条件に名前を付けて保存
  - [ ] 保存した条件の一覧表示・呼び出し
  - [ ] 保存条件の編集・削除
  - [ ] 共有リンクの生成（URLパラメータで条件を再現）

### FR-007: データ更新通知

- **優先度**: P2-Medium
- **説明**: 保存した検索条件に一致する新規法人を通知する
- **受入基準**:
  - [ ] 保存条件に新規マッチする法人が追加された場合にメール通知
  - [ ] 通知頻度の設定（日次/週次/月次）
  - [ ] 新規追加企業の差分リスト表示

### FR-008: ユーザー認証・課金

- **優先度**: P0-Critical
- **説明**: ユーザー登録・ログイン・サブスクリプション課金
- **受入基準**:
  - [ ] メール + パスワードでのユーザー登録・ログイン
  - [ ] Google OAuth ログイン対応
  - [ ] Stripe月額サブスクリプション課金
  - [ ] 無料プラン（機能制限付き）
  - [ ] プラン変更・解約の自動処理
  - [ ] 請求書・領収書の発行

### FR-009: ダッシュボード

- **優先度**: P1-High
- **説明**: 利用状況と保存リストの管理
- **受入基準**:
  - [ ] 今月のダウンロード件数 / 上限の表示
  - [ ] 保存した検索条件の一覧
  - [ ] ダウンロード履歴
  - [ ] プラン情報と使用率

### FR-010: 地域統計ダッシュボード

- **優先度**: P2-Medium
- **説明**: 業種×地域の企業分布を可視化する
- **受入基準**:
  - [ ] 都道府県別の企業数ヒートマップ（日本地図）
  - [ ] 業種別の企業数棒グラフ
  - [ ] 業種×地域のクロス集計表
  - [ ] 設立年の推移グラフ
  - [ ] データのスナップショット保存

## 3. 非機能要件

### NFR-001: パフォーマンス

| 指標 | 目標値 |
|------|--------|
| 検索結果表示 | < 3s（500万件からの絞り込み） |
| ページ初期表示 | < 2s |
| CSV/Excelダウンロード（1,000件） | < 5s |
| CSV/Excelダウンロード（10,000件） | < 30s |
| 同時検索ユーザー | 500+ |

### NFR-002: データ鮮度

| 項目 | 頻度 |
|------|------|
| gBizINFO同期 | 日次バッチ（差分取得） |
| 国税庁全件データ同期 | 月次（CSV一括インポート） |
| 市区町村マスタ更新 | 年次（総務省データ） |
| 産業分類マスタ更新 | 随時（総務省改定時） |

### NFR-003: セキュリティ

| 項目 | 対策 |
|------|------|
| API通信 | 全てHTTPS |
| ユーザーデータ | Supabase RLS + 暗号化 |
| 決済 | Stripe PCI DSS準拠 |
| CSRF/XSS | Next.js標準 + CSP |
| ダウンロード制限 | プラン別レートリミット |

### NFR-004: スケーラビリティ

| 項目 | 対応 |
|------|------|
| データ量 | 500万+ 法人レコード → PostgreSQLフルテキスト検索 + インデックス |
| ユーザー増加 | Vercel自動スケール + Supabase接続プール |
| 検索高速化 | PostgreSQL GINインデックス + Materialized View |
| ダウンロード | 大量ダウンロードはサーバーサイド非同期生成 |

## 4. 技術制約

### TC-001: 技術スタック

> **方針**: 無料枠を最大限活用。データソースが無料のため、固定費を極限まで抑える。

| カテゴリ | 技術 | 無料枠 | 用途 |
|----------|------|--------|------|
| **フレームワーク** | Next.js 15 (App Router) | - | フロントエンド + API Routes |
| **言語** | TypeScript 5.3+ | - | strict mode |
| **バックエンド/DB** | Supabase | 500MB DB, 50K MAU | DB・認証 |
| **デプロイ** | Vercel | 100GB帯域/月 | ホスティング |
| **DNS/CDN** | Cloudflare | 無制限 | DNS・WAF |
| **認証** | Clerk | 10,000 MAU | ユーザー認証 |
| **決済** | Stripe | 売上の3.6% | サブスクリプション |
| **メール** | Resend | 3,000通/月 | 通知メール |
| **エラー監視** | Sentry | 5,000イベント/月 | エラートラッキング |
| **gBizINFO API** | 経済産業省 | **無料** | 企業データ取得 |
| **国税庁API** | 国税庁 | **無料** | 法人番号・基本情報 |

### TC-002: gBizINFO REST API 仕様

| 項目 | 詳細 |
|------|------|
| ベースURL | `https://info.gbiz.go.jp/hojin/v1/hojin` |
| 認証 | アクセストークン（申請制、無料） |
| レスポンス | JSON |
| レートリミット | 非公開（過度なアクセスは制限） |
| 検索パラメータ | 法人名、法人番号、所在地（都道府県コード、市区町村コード）、業種 |
| 取得可能項目 | 法人番号、法人名、所在地、代表者、資本金、従業員数、営業品目、事業概要、HP URL、設立年、法人種別 |
| 次期版 | REST API v2 公開予定 |

### TC-003: 国税庁法人番号 Web-API 仕様

| 項目 | 詳細 |
|------|------|
| ベースURL | `https://api.houjin-bangou.nta.go.jp` |
| 認証 | アプリケーションID（申請制、無料） |
| レスポンス | XML / CSV |
| 検索方式 | 法人番号指定、法人名指定、期間指定 |
| 取得項目 | 法人番号、商号、所在地、法人種別、変更履歴 |
| 全件データ | CSV一括ダウンロード（都道府県別、月次更新） |

### TC-004: データ統合戦略

```
[gBizINFO REST API]                  [国税庁 全件CSV]
  │ (業種・代表者・資本金・HP等)         │ (最新法人名・所在地)
  │                                       │
  ▼                                       ▼
[データ取得バッチ (日次)]           [月次インポートバッチ]
  │                                       │
  └──────────┬──────────────────────────┘
             │
             ▼
    [法人番号で名寄せ・統合]
             │
             ▼
    [companies テーブル (Supabase)]
             │
             ▼
    [全文検索インデックス (GIN)]
             │
             ▼
    [ユーザー検索・ダウンロード]
```

### TC-005: DB容量の考慮

| データ | 想定レコード数 | 想定サイズ |
|--------|-------------|-----------|
| 法人データ | 約500万件 | 約2〜3GB |
| インデックス | - | 約1〜2GB |
| ユーザーデータ | 〜10,000 | 数MB |
| 合計 | - | 約3〜5GB |

> **注意**: Supabase Free プランは500MB。初期段階で **Supabase Pro（$25/月）** が必須。または外部のPostgreSQLサービス（Neon、Railway等）の利用を検討。

### TC-006: 500万件対応の検索最適化

| 手法 | 説明 |
|------|------|
| **Materialized View** | 業種×都道府県の事前集計ビュー（件数表示高速化） |
| **GINインデックス** | 法人名・事業概要の全文検索用 |
| **パーティショニング** | 都道府県別テーブルパーティション（検索範囲限定） |
| **カーソルページネーション** | OFFSET不使用、keyset paginationで大量データ対応 |
| **検索結果キャッシュ** | 同一条件の検索結果を短時間キャッシュ（Upstash Redis） |

## 14. リスクと対策

|--------|------|------|
