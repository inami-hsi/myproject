# BCP自動診断ツール - 機能要件書

## 1. プロジェクト概要

### 1.1 プロジェクト名
BCP自動診断・ひな形自動生成ツール（BCP Diagnosis & Template Generator）

### 1.2 目的
生損保保険代理店業の企業が、顧客および自社のBCP（事業継続計画）を**オールハザード**で包括的に診断し、
政府様式準拠の「事業継続力強化計画」ひな型を自動生成するWebアプリケーション。

### 1.3 対象ユーザー
- 保険代理店の営業スタッフ
- 保険代理店の企画・経営企画部門
- 対象企業の経営層・BCP責任者

### 1.4 提供方法
- **Web アプリケーション**（SaaS型）
- 既存Next.jsプロジェクト内に組み込み
- ブラウザアクセスのみで利用可能

#### 2.1.2 マルチステップ Wizard UI (`/bcp-tool/wizard`)

**全8ステップ構成**：

##### **Step 1: 基本情報入力**
- 必須入力項目：
  - 会社名
  - 業種（ドロップダウン：製造・小売・IT・保険・サービス等）
  - 従業員数
  - 主要拠点数
  - 代表者名
  - BCP責任者名
  - 責任者連絡先（電話・メール）
  - 策定日
  - 対象範囲（全社/特定部門）

- バリデーション：
  - 必須項目のチェック
  - メールアドレス形式の検証
  - 電話番号フォーマット

---

##### **Step 2: オールハザード評価**
- ハザード選択（チェックボックス）：
  - ☐ 地震
  - ☐ 水害（洪水・津波）
  - ☐ 停電・インフラ障害
  - ☐ パンデミック
  - ☐ サイバー攻撃
  - ☐ 火災
  - ☐ サプライチェーン災害

- 各ハザードごとに「現在の対応度」をスライダーで入力（1〜10）

- 複数ハザード同時選択時に「複合シナリオ」オプションを表示
  - 例：「地震 + 停電 + サイバー」の同時発生を考慮

- バリデーション：
  - 最低1つ以上のハザード選択が必須

---

##### **Step 3: 重要業務の特定**
- テーブル形式での追加・編集・削除機能
  - 業務名（テキスト）
  - 担当部門（テキスト）
  - 優先度（P1/P2/P3から選択）
  - 目標復旧時間（RTO）（入力：時間）
  - 目標復旧レベル（RPO）（入力：時間）

- 各業務ごとに「ハザード別対応方法」を入力
  - 行動：「詳細を入力」ボタンクリック → モーダルが開く
  - モーダル内：
    ```
    業務：「顧客対応・契約処理」
    ├─ 地震時：[対応方法入力]
    ├─ 停電時：[対応方法入力]
    ├─ サイバー時：[対応方法入力]
    └─ ...（全選択ハザード分）
    ```

- デフォルト行の用意（編集可能）：
  - 「顧客対応・契約処理」（営業部門）P1 4時間
  - 「保険金支払処理」（経理部門）P1 8時間
  - 「システム運用」（IT部門）P1 2時間

- バリデーション：
  - 最低1つは重要業務が必須
  - RTOは数値のみ
  - 優先度の必須選択

---

##### **Step 4: 予防策・対応策入力**
- ハザード別のアコーディオン形式（デフォルト：閉じた状態）
  - 例：「▶ 地震対策」「▶ 停電対策」など

- 各ハザードごとに以下を入力：
  - **予防策** (テキストエリア)
    - 複数行入力可能
    - テンプレート：「耐震補強」「什器固定」「BCP訓練」など提示
  - **対応策** (テキストエリア)
    - 複数行入力可能
    - テンプレート：「初動マニュアル」「テレワーク対応」「代替拠点」など提示

- 「テンプレートを挿入」ボタン
  - クリックでハザード別の推奨テンプレート例を表示・選択可能

- バリデーション：
  - Step 2で選択されたハザードのみ入力フォームを表示
  - 最低1つは予防策または対応策が必須

---

##### **Step 5: 復旧戦略・リソース**
- フォーム項目：
  - バックアップ取得頻度（ドロップダウン：毎日/毎週/毎月）
  - バックアップ保管場所（複数選択可）：
    - ☐ 複数拠点
    - ☐ クラウド（AWS/Azure/Google Cloud）
    - ☐ 外部データセンター
  - 代替拠点の有無（Yes/No）
    - Yes選択時：住所、設備概要を入力
  - 重要データ保有種類（テキスト）
  - データバックアップ手法（テキストエリア）
  - BCPに配分可能な年間予算（数値：万円）
  - 外部リソース利用状況（テキストエリア）
    - クラウドサービス、物流パートナー等

- バリデーション：
  - 予算は0以上の数値のみ
  - 代替拠点選択時は住所必須

---

##### **Step 6: 組織体制・責任者**
- テーブル形式での責任者設定：
  | 役割 | 担当者名 | 連絡先（電話） | メール |
  |------|---------|----------------|--------|
  | BCP推進責任者 | [入力] | [入力] | [入力] |
  | 初動対応チーム長 | [入力] | [入力] | [入力] |
  | システム復旧責任者 | [入力] | [入力] | [入力] |
  | 社員安全管理責任者 | [入力] | [入力] | [入力] |
  | 外部通信責任者 | [入力] | [入力] | [入力] |
  | （その他追加可） | | | |

- 行の追加・削除機能
- バリデーション：
  - BCP推進責任者は必須
  - メールアドレス形式の検証
  - 電話番号フォーマット

---

##### **Step 7: 教育・訓練計画**
- フォーム項目：
  - 年間訓練回数（入力：回数）
  - 訓練形式（複数選択可）：
    - ☐ 机上演習
    - ☐ 実地訓練
    - ☐ 複合シナリオ訓練
  - 新入社員教育：Yes/No
  - 訓練予定（月をチェックボックスで選択）：
    - ☐ 4月 ☐ 5月 ☐ 6月 ☐ 7月 ☐ 8月 ☐ 9月 ☐ 10月 ☐ 11月 ☐ 12月 ☐ 1月 ☐ 2月 ☐ 3月
  - 訓練評価方法（テキストエリア）

- バリデーション：
  - 訓練回数は0以上の整数
  - 最低1つは訓練形式を選択

---

##### **Step 8: レビュー & 生成**
- 入力内容の最終確認
  - 各ステップの概要を表示（折りたたみ可能）
  - 各セクションの「編集」ボタンで戻る機能

- 生成ボタン（2つ）：
  - 📄 **Word で生成** ボタン
  - 📋 **PDF で生成** ボタン

- 生成中の表示：
  - ローディングスピナー + 「BCP ひな形を生成中...」メッセージ
  - 3秒～10秒程度で完了

- 生成完了後：
  - ブラウザの自動ダウンロード
  - ファイル名：`BCP_{会社名}_{生成日付}.{docx|pdf}`
  - 成功メッセージを表示

- エラーハンドリング：
  - バリデーションエラー時は該当ステップでエラーメッセージ表示
  - サーバーエラー時はモーダルで通知

#### 2.2.2 BCP 生成エンジン（`bcp-generate`）
**入力**：
- フォームデータ（JSON）
  ```json
  {
    "basicInfo": {...},
    "hazards": {...},
    "criticalBusiness": [...],
    "strategies": {...},
    "recovery": {...},
    "organization": [...],
    "training": {...}
  }
  ```

**処理**：
1. バリデーション（Zod schema）
2. テンプレートエンジンにデータをバインド
3. AI補強（Claude API を使い対策案を生成）
4. Word ファイル生成（docx ライブラリ）
5. PDF 変換（jsPDF または外部サービス）
6. AWS S3 へのアップロード（一時保存）

**出力**：
- Word ファイル (.docx)
- PDF ファイル (.pdf)

---

#### 2.2.3 診断エンジン（`bcp-diagnose`）
**機能**：
- ハザード別の脅弱性スコア計算
- 複合ハザードシナリオ生成（AI使用）
- 対策の優先度付け自動提案
- 診断レポート JSON

**出力**：
```json
{
  "bcpScore": 65,
  "hazardScores": {
    "地震": 70,
    "水害": 55,
    "停電": 40,
    ...
  },
  "vulnerabilities": [...],
  "recommendations": [...]
}
```

---

#### 2.2.4 AI統合（Claude API）
**用途**：
- 入力内容から「想定被害シナリオ」を自動生成
- ハザード別の「推奨対策文言」の提示
- 複合ハザード時のリスク分析

**API キー**：`.env.local` に保管

### 2.3 データベース機能

#### 2.3.1 新規テーブル （Prisma schema）

**`BCPDiagnosis` テーブル**
```prisma
model BCPDiagnosis {
  id            String   @id @default(cuid())
  userId        String   @db.VarChar(255)
  companyName   String   @db.VarChar(255)
  industryType  String   @db.VarChar(100)
  employeeCount Int
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
  formData      Json     // 全フォームデータを JSON で保管
  riskScore     Int?
  status        String   @default("draft")  // draft, completed, archived
  pdfUrl        String?  @db.Text
  wordUrl       String?  @db.Text
  expiresAt     DateTime // ファイル削除期限（30日）
}
```

**`BCPTemplate` テーブル**
```prisma
model BCPTemplate {
  id          String   @id @default(cuid())
  hazardName  String   @unique
  preventions String[] // 予防策のテンプレート配列
  responses   String[] // 対応策のテンプレート配列
  updatedAt   DateTime @updatedAt
  createdAt   DateTime @default(now())
}
```

## 5. 制約事項
