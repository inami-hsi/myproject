# BCP自動診断ツール - 非機能要件書

## 1. パフォーマンス要件

### 1.1 応答時間
| 操作 | 目標応答時間 |
|------|-----------|
| ページロード（初期） | < 2 秒 |
| Step 遷移 | < 500 ms |
| バリデーション実行 | < 200 ms |
| 診断結果計算 | < 3 秒 |
| Word/PDF 生成 | 3～10 秒 |
| ダッシュボード読み込み | < 2 秒 |

### 1.2 スループット
- **同時ユーザー数**：100 concurrent users
- **API リクエスト数**：1000 requests/min（バースト対応）
- **データベース接続池**：20～50 接続

### 1.3 キャッシング戦略
- **フロントエンド**：
  - React コンポーネント（React.memo）
  - Next.js Static Generation（/bcp-tool ランディング）
  - Service Worker（オフライン対応）

- **バックエンド**：
  - API レスポンスキャッシュ（Redis）：TTL 5分
  - データベースクエリキャッシュ：TTL 1時間
  - テンプレートキャッシュ：永続化

- **CDN**：
  - 静的ファイル（JS/CSS/画像）を Vercel/Cloudflare CDN 配信

## 3. 信頼性・可用性要件

### 3.1 可用性（Availability）
- **目標稼働率**：99.5%（月間ダウンタイムの上限：3.6 時間）
- **RTO（目標復旧時間）**：< 4 時間
- **RPO（目標復旧地点）**：< 1 時間

### 3.2 バックアップ・復旧
- **バックアップ頻度**：
  - データベース：毎日 深夜 0 時（差分）+ 毎週日曜 0 時（フル）
  - S3 ファイル：クロスリージョンレプリケーション（自動）

- **復旧手順**：
  - Point-in-Time Recovery：24 時間以内の任意ポイント
  - 災害復旧テスト：四半期 1 回以上実施

### 3.3 エラーハンドリング
- **503 Service Unavailable**：
  - API エラー時の自動リトライ（最大 3 回）
  - 指数バックオフ（1s, 2s, 4s）

- **5xx エラー応答**：
  - ユーザーへの詳細エラーメッセージ避退
  - 代わりに「ご不便をおかけしています。しばらく待ってからお試しください」

- **ネットワークタイムアウト**：
  - デフォルト：30 秒
  - ファイル生成API：60 秒

### 3.4 フォールバック
- **Word/PDF 生成フェール時**：
  - JSON 形式でのダウンロード提供
  - ク後日再生成できることをユーザーに表示

## 4. セキュリティ要件

### 4.1 認証・認可
**本ツールは認証不要**（個人情報非保存設計のため）

代わりに以下を実装：
- **入力データの即時処理・削除**
  - フォーム送信 → 処理 → ファイル生成 → ブラウザダウンロード → サーバー側データ削除
  - サーバーにユーザーデータを保持しない

- **セッション管理**：不要（ステートレス設計）

### 4.2 データ保護
- **転送中の暗号化**：
  - HTTPS/TLS 1.3 以上

- **保存時の暗号化**：
  - S3 一時ファイル：サーバー側暗号化（SSE-S3）
  - ファイル削除：生成直後 → ユーザーダウンロード → サーバー側から即削除

- **個人情報の非保存**：
  - 入力フォームデータ：メモリ上のみ処理
  - データベース：ユーザー・診断結果テーブル不要

### 4.3 入力検証・XSS 対策
- **バリデーション**：Zod/Yup による型安全な検証
- **XSS 対策**：
  - React の自動エスケープ
  - DOMPurify による危険な HTML 削除（編集コンテンツ）

- **SQL インジェクション対策**：
  - ORM（Prisma）による parameterized queries

- **CSRF 対策**：
  - CSRF トークン（POST/PUT/DELETE）

### 4.4 API セキュリティ
- **Rate Limiting**：
  - ユーザーあたり：100 requests/min
  - IP あたり：1000 requests/min

- **DDoS 対策**：
  - CloudFlare/AWS Shield Standard を使用

### 4.5 監査ログ
**個人ユーザー追跡ログは不要。以下のシステムログのみ記録**

- **ログ対象操作**：
  - API エラー（500, 403 等）
  - Word/PDF 生成の失敗
  - サーバーダウンタイム
  - 外部 API（Claude）の呼び出しエラー

- **ログ記録**：
  - CloudWatch ログ（シンプル構成）
  - 保持期間：30 日間
  - 含まれないデータ：ユーザー ID、IP アドレス、入力内容

## 7. コンプライアンス・ガバナンス

### 7.1 個人情報保護（簡素化）
**個人情報非保存設計のため、PDPC対応は最小限**

- **個人情報定義**：保存しない（入力中のメモリ上のみ、生成直後破棄）
- **保有データ**：
  - 入力フォーム：ブラウザメモリに一時保持（送信後削除）
  - Word/PDF：生成直後、ブラウザダウンロード後 S3 から削除
  - サーバー側：個人識別情報を記録しない

- **方針声明**：
  > 「本ツールはユーザーの個人情報を保存しません。入力データはひな形生成のため一時的に処理され、生成直後に完全に破棄されます。」
  
  このポリシーを利用規約に明記

### 7.2 情報セキュリティ
**ISMS認証は不要（個人情報データベース非構築）**

代わりに以下を実装：
- **セキュアコーディング**：
  - OWASP Top 10 対応（XSS/SQL Injection/CSRF 等）
  - 入力バリデーション（Zod schema 強化）
  - Content Security Policy（CSP）ヘッダ

- **セキュリティ監査**：年 1 回（簡略版、外部可選択）
  - コード静的解析（SonarQube など）
  - 依存関係脆弱性スキャン（npm audit, Snyk）

- **ペネトレーションテスト**：オプション（必須ではない）

### 7.3 アクセシビリティ
- **WCAG 2.1 Level AA 準拠**
  - カラーコントラスト比 4.5:1 以上
  - キーボードナビゲーション対応
  - スクリーンリーダー対応（aria-label 等）

### 7.4 利用規約・ポリシー
- **利用規約**：簡潔版（個人情報非保存を明記）
- **プライバシーポリシー**：最小限（ブラウザローカルストレージのみ）
- **免責事項**：生成されたひな形は参考資料であり、法的責任は負わない

## 9. クラウドインフラ要件

### 9.1 ホスティング環境
- **本番**：AWS ECS Fargate（Next.js コンテナ）
  - インスタンスサイズ：0.5 vCPU, 1 GB RAM（最小構成）
  - スケーリング：API リクエスト数に応じて自動スケール
  
- **ステージング**：同構成（ただし 1 インスタンスに縮小）

- **開発**：ローカル Docker Compose

### 9.2 データベース（オプション）
- **不要**：個人情報・診断履歴を保存しないため、RDS 不要
- **代替**：テンプレート管理のみが必要な場合、シンプル JSON/YAML を S3 に保存でも可
  - またはDockerコンテナ内に埋め込み

### 9.3 ストレージ
- **AWS S3**（診断結果ドキュメント一時保管）
  - 生成直後 → ユーザーダウンロード → サーバー側削除（1 時間以内）
  - Lifecycle Policy：1 時間後に自動削除
  - Versioning：不要

### 9.4 CI/CD
- GitHub Actions
  - `git push` → 自動テスト → Staging デプロイ
  - PR マージ → 手動承認 → 本番デプロイ
