## 0. プロジェクト概要（再掲）

### 背景
顧客が多数の保険商品・保険会社から最適な商品を選定する際、**保険業法で定められたロ方式**（顧客の意向に基づくプロセス）に従いながら、**合理的な推奨**を実現するWebベースのガイダンスシステム。

### ゴール
- 損害保険4種目 × 5社
- 生命保険8種目 × 7社
- 計12種目・12社の保険商品から、ユーザー回答に基づき、**最適な3社を推奨**

### スコープ（Phase 1-2）
✅ 要件定義完了  
✅ フロー設計完了（損保詳細、生保概要）  
✅ デザインシステム完了  
✅ ワイヤーフレーム完了  
⏳ 次: 実装フェーズ開始（Phase 3）

### スコープ外（今後検討）
- 保険料 API 連携（初版は目安データのみ）
- 生命保険詳細フロー（損保実装後に設計）
- モバイルアプリ化（Web ファースト）

### 4.3 `src/lib/types.ts` - TypeScript型定義

```typescript
export interface InsuranceCompany {
  id: string;
  name: string;
  category: 'loss' | 'life';
  url: string;
  scoring: Record<string, number>;
}

export interface Question {
  step: number;
  question: string;
  type: 'single-select' | 'multi-select' | 'slider';
  options: QuestionOption[];
}

export interface QuestionOption {
  id: string;
  label: string;
  score?: Record<string, number>;
}

export interface UserAnswers {
  insuranceType: string; // 'auto-loss', 'fire', 'life-medical'等
  answers: Record<number, string | string[]>; // step番号 → 選択肢ID
}

export interface Recommendation {
  rank: 1 | 2 | 3;
  companyId: string;
  companyName: string;
  productName: string;
  estimatedPremium?: string;
  reasoning: {
    summary: string; // 1-2行の簡潔版
    detailed: string; // 段落分けの詳細版
  };
  matchScore: number; // 0-100
}
```

## 6. 重要な実装判定点（デシジョンポイント）

### D1: 保険料見積データの取り扱い
**決定**: 初版は「目安」データ（ダミー）  
**代替手段**: 将来、各保険会社 API 連携時に自動更新

実装時: `src/data/premiums.json`（目安データセット）にハードコード

### D2: ユーザー回答の永続化
**決定**: 初版は LocalStorage（ブラウザローカル）  
**代替手段**: 将来、Database（Supabase等）に保存

実装時: Zustand でメモリ管理 + useEffect で LocalStorage 同期

### D3: 生命保険フロー詳細設計の時期
**決定**: 損保フロー実装**基本完成後**（Phase 4中盤）に詳細設計  
**理由**: 同じチームで技術知見を引き継げる、ユーザーフィードバックを反映できる

実装時: フローの共通ライブラリ化（DRY原則）

### D4: モバイルアプリ化
**決定**: Web ファースト（PWA化は オプション）  
**代替手段**: 将来、React Native/Flutter で native app 化

実装時: `viewport`, `manifest.json`, Service Worker （基本設定のみ）

## 9. 品質基準

### 機能要件
- [ ] 12種目全てでヒアリング→推奨をできる
- [ ] 推奨理由が4セクション（マッチ度・比較・利用シーン・保険料）で明確
- [ ] 3社の比較表が並行表示される

### 非機能要件
- [ ] Lighthouse Performance 90+
- [ ] Accessibility 95+
- [ ] WCAG 2.1 AA 準拠
- [ ] モバイル（375px）～ Desktop（1920px）対応
- [ ] 初期表示 1.8秒以内

### ユーザー体験要件
- [ ] 5分でヒアリング完了 → 推奨結果閲覧可能
- [ ] 全文字が読みやすい（最小フォント 12px）
- [ ] タップターゲット最小 44px

## 10. よくある質問（FAQ）

### Q1: 生命保険の詳細フロー設計は必要？
