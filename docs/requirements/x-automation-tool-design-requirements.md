## 1. 美的方向性（Aesthetic Direction）

### 1.1 トーン

- [x] **industrial-utilitarian** - インダストリアル、機能性重視、高密度情報表示

X自動化ツールは「効率」と「制御」が最重要。
マーケターやインフルエンサーが大量のアクション・データを一目で把握し、
素早く操作できる機能性重視のダッシュボードが求められる。

### 1.2 ムード

- [x] **professional**（プロフェッショナル）
- [x] **bold**（大胆）

ビジネスツールとしての信頼性と、X（旧Twitter）ブランドに
通じるダークモードベースのモダンなUI。

### 1.3 参照サイト

| サイト名 | URL | 参考ポイント |
|---------|-----|-------------|
| XTEP | https://xtep.tools | X自動化ツールの標準UI/UX |
| Buffer | https://buffer.com | SNS管理ダッシュボードの情報設計 |
| Hootsuite | https://hootsuite.com | 多機能SNSツールのナビゲーション設計 |
| X Analytics | https://analytics.twitter.com | X公式の分析画面のデータ表示 |

### 1.4 禁止パターン（AI Slop回避）

以下は**使用禁止**:

- [x] Inter, Roboto, Arial等のジェネリックフォント
- [x] 紫グラデーション on 白背景
- [x] 予測可能なレイアウトパターン
- [x] 過剰なドロップシャドウ
- [x] Space Grotesk（AI生成の典型）
- [x] cookie-cutter design
- [x] ライトモードのみのSNSツール

### 1.5 差別化ポイント

```
「X運用のコックピット」— パイロットが計器盤を見るように、
API利用量・送信キュー・キャンペーン状況を一画面でリアルタイムに把握。
レートリミットのゲージバーとキュー処理のライブインジケーターで
「今何が動いているか」を常に感じられる操作体験。
```

## 6. アニメーション方針

### 6.1 基本ルール

| ルール | 詳細 |
|--------|------|
| 最大デュレーション | 200ms（インタラクション系） |
| トランジションプロパティ | transform, opacity のみ（compositor props） |
| イージング | ease-out（出現）、ease-in（消失） |

### 6.2 使用シーン

| シーン | アニメーション | ライブラリ |
|--------|------------|-----------|
| API残量ゲージ更新 | 幅アニメーション (width transition) | CSS transition |
| 送信ログ追加 | スライドイン + フェードイン | motion/react |
| トリガーチップ選択 | スケール + 色変化 | CSS transition |
| ステータスバッジ変化 | フェード切替 | CSS transition |
| ページ間トランジション | フェードイン | motion/react |
| モーダル表示 | スケールアップ + フェードイン | motion/react |
| 送信成功通知 | スライドイン (右下) + 自動消去 | motion/react |
| データリフレッシュ | ローテーション（リフレッシュアイコン） | CSS animation |

### 6.3 禁止アニメーション

- layout プロパティ（width, height, top, left）のアニメーション
- 3秒以上の連続アニメーション
- 自動再生されるアニメーション（prefers-reduced-motion 考慮）
- ダッシュボードでの過度なアニメーション（操作効率を阻害しない）

## 8. 追加要件

### 8.1 API利用量ゲージ

- DM残量（X/100）とリプライ残量（X/2,400）をプログレスバーで表示
- 残量に応じた色変化: 十分(グリーン) → 注意(アンバー) → 危険(レッド)
- リセットまでの残り時間をカウントダウン表示
- tabular-nums で数字が揺れないように固定幅
- サイドバーにもコンパクト版を常時表示

### 8.2 送信ログのリアルタイム表示

- 新しいログが追加されたらスライドインアニメーション
- ステータスアイコン（成功 ✓ / 失敗 ✗ / キュー ⏳ / レート制限 ⚠）
- ログのフィルタリング（種別・ステータス・日時）
- 詳細表示（APIレスポンス確認）

### 8.3 トリガー設定のUX

- トリガー条件はチップ型トグルボタンで直感的に選択
- 条件の組み合わせ（AND/OR）を視覚的に表示
- テンプレート変数は入力中にオートコンプリート提案

### 8.4 キャンペーン設定ウィザード

- 3種のキャンペーンタイプをカード型で選択
- ステップ形式で進行（プログレスバー付き）
- 各ステップにプレビュー表示

### 8.5 メールデザイン

- react-email でHTML/テキスト両対応
- ブランドカラー統一（Xブルーベース）
- モバイルレスポンシブ
- CTAボタンは大きく、タップしやすく
