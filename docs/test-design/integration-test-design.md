## 2. データベーステスト戦略

### 2.1 テスト用DB管理

```
テスト開始前:
  1. Docker Compose でPostgreSQLを起動
  2. prisma migrate deploy でスキーマ適用
  3. シードデータ投入

各テストスイート前:
  4. テーブルクリーンアップ (TRUNCATE CASCADE)
  5. テスト固有のシードデータ投入

テスト終了後:
  6. Docker Compose停止（オプション: CI環境では自動）
```

// tests/integration/helpers/cleanup.ts
import { PrismaClient } from "@prisma/client";

const prisma = new PrismaClient();

export async function cleanupDatabase() {
  // 依存関係の順序でTRUNCATE
  await prisma.$executeRawUnsafe(`
    TRUNCATE TABLE "Comment" CASCADE;
    TRUNCATE TABLE "TagOnTask" CASCADE;
    TRUNCATE TABLE "TaskDependency" CASCADE;
    TRUNCATE TABLE "Task" CASCADE;
    TRUNCATE TABLE "Milestone" CASCADE;
    TRUNCATE TABLE "Tag" CASCADE;
    TRUNCATE TABLE "Project" CASCADE;
    TRUNCATE TABLE "User" CASCADE;
  `);
}
