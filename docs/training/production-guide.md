## 🎥 第1章：撮影ガイド

### 1.1 カメラ設定（スマートフォン・一眼レフ共通）

#### **解像度・フレームレート**

| 項目 | 推奨値 | 説明 |
|------|--------|------|
| **解像度** | 1920×1080 (Full HD) | YouTubeガイドライン標準 |
| **フレームレート** | 24fps | 映画的表現、軽い容量 |
| **ビットレート** | 8-12 Mbps | バランスの取れた品質 |
| **コーデック** | H.264 | 編集・配信最適化 |

**設定手順**：
1. スマートフォン：設定 → カメラ → 解像度 → 1080p選択
2. 一眼レフ：メニュー → 動画記録設定 → 1920×1080 / 24p

#### **ホワイトバランス**

| 光源 | 色温度 | 設定値 |
|------|--------|--------|
| 自然光（晴天） | 5500K | 自動 または Daylight |
| 蛍光灯 | 4000K | Fluorescent |
| LED撮影用ライト | 5600K | Daylight |
| オフィス照明 | 3200-4000K | 手動で3200K ~ 4000K |

**ホワイトバランス調整手順**：
- スマートフォン：タップ → WB（ホワイトバランス）→ 光源に合わせる
- 一眼レフ：WBボタン長押し → 光源選択 / グレーカード撮影で自動

#### **露出・ISO・シャッタースピード**

```
概念：
- 露出 = 光量調整
- ISO = 光への感度
- シャッタースピード = センサー開放時間
```

| 設定 | 値 | 用途 |
|-----|-----|------|
| **ISO** | 100-400 | スタジオ（照明充実） |
| **ISO** | 400-1600 | 普通のオフィス |
| **ISO** | 1600-3200 | 暗いシーン（推奨しない） |
| **シャッタースピード** | 1/48秒 | 24fps時の標準 |

**設定手順**：
1. ISO: 「低いほど高品質」を原則に、最低限必要な値を選択
2. シャッタースピード: 24fpsの場合は1/48秒（フレームレートの2倍が目安）
3. 露出: 顔が過度に明るくなければOK

### 1.3 マイク・音声レベル設定

#### **マイク選定基準**

| マイク種別 | コスト | 推奨 | 用途 |
|-----------|--------|------|------|
| **内蔵マイク** | ¥0 | ✗ | 廃止（ノイズ多） |
| **ラベリアマイク** | ¥5,000 | ◎ | ナレーター・司会 |
| **ショットガンマイク** | ¥10,000 | ◎ | 環境音キャプチャー |
| **USBコンディンサー** | ¥15,000 | ◎ | ナレーション後録音 |
| **プロ用XLRマイク** | ¥30,000+ | ◎◎ | スタジオ録音 |

#### **推奨セットアップ：ラベリアマイク + オーディオインターフェース**

```
[ラベリアマイク]
      ↓
[3.5mmケーブル]
      ↓
[オーディオインターフェース] ← 推奨: Focusrite Scarlett 2i2, Behringer U-Phoria
      ↓ (USB-C / USB)
[カメラ or スマートフォン]
```

**推奨製品**：
- マイク：Audio-Technica AT831b（¥8,000）
- インターフェース：Focusrite Scarlett Solo（¥10,000）

#### **音声レベル設定**

**重要ルール**：
```
ピークメーター最大値: -6dB ~ -3dB
避けるべき値：
  - -12dB以下：小さすぎて後処理困難
  - 0dB以上：クリップ（音割れ）発生
```

**レベルチェック手順**：

1. **カメラ・スマートフォン側設定**：
   - スマートフォン：ボリュームアイコン → マック音量 50-70%
   - 一眼レフ：メニュー → 動画 → マイク音量 → -6 ~ -3

2. **スマートフォン app レベル確認**：
   - 無料app「Decibel X」をインストール
   - タレント会話時 60-70dB を目安

3. **テスト撮影**：
   - 30秒～1分テスト撮影
   - 再生して聞き確認
   - ノイズがあれば マイク位置・感度を調整

#### **ノイズ対策**

| ノイズ種別 | 原因 | 対策 |
|-----------|------|------|
| **65dB以上（風切音）** | 風・エアコン | USB ウインドスクリーン装着、エアコン停止 |
| **ハム音（50-60Hz）** | 電源干渉 | マイクケーブル位置変更、接地確認 |
| **キーンという音** | フィードバック | スピーカーオフ、マイク距離調整 |
| **カリカリ音** | ケーブル触接 | ケーブルテープ固定、短くする |

### 1.4 タレント（ナレーター）のポジション・ジェスチャー指示

#### **ポジショニング**

```
              Camera
                ↓
           [Standard]
                ↓
         Eye Level (目ン高さ)
           ← 1.5-2m →
                ↓
            [Talent]
               身体
        (from feet to head)
```

**距離・角度設定**：

| 要素 | 値 | 理由 |
|-----|-----|------|
| **カメラ距離** | 1.5-2m | 脚部から頭部全体が入る |
| **カメラ高さ** | 目ン高さ | 自然な視線（見下ろす・仰ぎ見ない） |
| **カメラ角度** | 正面 or 15°斜め | 正面が基本、スピーチは15°は OK |
| **タレント位置** | 背景から1.5m+ | 背景ぼかし（ボケ）を出す |

#### **フレーミング（顔の構図）**

| フレーム種別 | 頭部 | 肩 | 腕 | 用途 |
|-----------|------|-----|-----|------|
| **Close-up** | フレーム内 | × | × | 感情・表情重視 |
| **Medium** | フレーム内 | ◎ | ◎ | 標準（講義） |
| **Wide** | フレーム内 | ◎ | ◎ | 全身＋環境 |

**推奨：Medium（肩から上）** - バランスよく視線を集中

#### **ジェスチャー・表情ガイド**

**顔の向き**：
- ✓ 正面（カメラ正対）
- ✓ 15°左向き（やや視線外して、リラックス感）
- ✓ 15°右向き（やや視線外して、視線誘導）

**禁止**：
- ✗ 横向き（45°以上）
- ✗ 見下ろし
- ✗ スマートフォン見ながら

**手ジェスチャー**：
| ジェスチャー | 効果 | 使用シーン |
|----------|------|---------|
| **両手ジェスチャー** | 強調・注意喚起 | ポイント説明 |
| **片手ジェスチャー** | ナチュラル | 標準的な説明 |
| **指差し** | 具体化・指摘 | リスク指摘 |
| **頷き** | 同意・確認 | リスナー共感 |

**禁止ジェスチャー**：
- ✗ 腕を組む（防御的）
- ✗ 手をポケットに（不安定）
- ✗ びっくり型上げ具合（大げさ）

**表情ガイド**：

```
表情レベル（0-5段階）

Lv 0: 無表情（静寂、重い雰囲気）→ 使用禁止
Lv 1: ニュートラル（リラックス、プロフェッショナル）→ 基本フェース
Lv 2: 穏やか微笑（親しみやすさ）→ 推奨フェース
Lv 3: 温かい笑顔（親近感・信頼）→ ウェルカムシーン
Lv 4: 大笑い（楽しさ・喜び）→ 成功時・祝い時
Lv 5: オーバー笑い（不自然）→ 使用禁止
```

**推奨表情**：Lv 2-3（穏やかな微笑～温かい笑顔）

#### **話し方ガイド**

| 要素 | 推奨値 | 説明 |
|-----|--------|------|
| **話速** | 100-120 words/min | 標準的なプレゼンテーション速度 |
| **トーン** | 中高音（明るい） | プロフェッショナルで親しみやすい |
| **抑揚** | あり（単調避け） | 重要ポイントで声を上げる |
| **一文** | 15-20秒 | 短文で理解促進 |
| **句読点** | 明確（2-3秒のポーズ） | 視聴者の理解時間確保 |

**NG例**：
- ✗ 120 words/min以上（話し急ぎ）
- ✗ 単調なお経のような話し方
- ✗ 「えっと」「あの」などの呟き

## ✂️ 第2章：編集ソフト別ワークフロー

### 2.1 DaVinci Resolve（無料版）での実装手順

**推奨理由**：
- 無料版でも機能豊か
- プロレベル色補正・字幕対応
- 初心者～プロまで対応
- 推定制作時間：1本あたり 3-5時間

#### **セットアップ**

```bash
# 1. ダウンロード
https://www.blackmagicdesign.com/products/davinciresolve

# 2. インストール
- Windows / Mac / Linux 対応
- メモリ: 16GB以上推奨
- GPU: NVIDIA/AMD あると高速化

# 3. 起動
- New Project で解像度 1920×1080、フレームレート 24fps 設定
```

#### **基本ワークフロー（4ステップ）**

##### **Step 1: プロジェクト設定**

```
File → New Project
  ├─ Project Name: Module_1-1_BCP_Basics
  ├─ Resolution: 1920×1080 (DCI)
  ├─ Frame Rate: 24fps
  ├─ Codec: H.264
  └─ Color Space: Rec.709
```

##### **Step 2: 素材インポート**

```
Media Pool（左下）
  ├─ Drag & Drop: 動画ファイル（mp4/mov）
  ├─ フォルダ作成: Module_1 / Footage / Audio / Graphics
  └─ オーガナイズ: 素材を見やすく整理
```

**推奨フォルダ構成**：
```
Bin (プロジェクト内フォルダ)
├── 01_RAW_VIDEO（撮影素材）
├── 02_NARRATION（ナレーション音声）
├── 03_GRAPHICS（テロップ・ロゴ）
├── 04_AUDIO_BG（BGM・効果音）
└── 05_SEQUENCE（編集済み動画）
```

##### **Step 3: タイムラインに配置**

```
[Edit] タブ → Timeline をドラッグ
  ├─ V1（Video 1）: メイン映像
  ├─ V2（Video 2）: テロップ・キー合成用
  ├─ A1（Audio 1）: ナレーション
  ├─ A2（Audio 2）: BGM
  └─ A3（Audio 3）: 効果音
```

**操作手順**：
1. Media Pool から動画ドラッグ → Timeline の V1 にドロップ
2. トリミング（In/Out点設定）で不要部分削除
3. トランジション追加（Dissolve など）

##### **Step 4: 色補正・エクスポート**

###### **色補正（簡易版）**

```
[Color] タブ
  ├─ Curves: 白黒バランス調整
  ├─ Saturation: 彩度調整
  ├─ Temp（色温度）: 暖色/寒色調整
  └─ Hue（色相）: 全体色トーン調整
```

**実装例**：
- 明るすぎる映像 → Curves で黒点・白点調整
- 色かぶり → Temp / Hue で色温度調整

###### **エクスポート設定**

```
File → Export → Add to Render Queue
  ├─ Format: MP4
  ├─ Codec: H.264
  ├─ Bitrate: 8-12 Mbps
  ├─ Audio: 128 kbps, 48 kHz
  └─ チェック: Render Individual Clips チェック外す
```

#### **高度なワークフロー（追加機能）**

##### **字幕追加（DaVinci Resolve字幕機能）**

```
[Edit] タブ → Timeline → Inspector（右上）
  ├─ Text+ をタイムラインに挿入
  ├─ テキスト入力
  ├─ フォント・サイズ・色設定
  └─ 図形マスク（字幕背景）追加
```

**字幕タイミング調整**：
- ナレーション開始 = 字幕開始
- ナレーション終了の0.1秒前 = 字幕終了

##### **キーフレーム・アニメーション**

```
Inspector → Transform
  ├─ Position（移動）
  ├─ Scale（拡大縮小）
  ├─ Rotation（回転）
  └─ Opacity（透度）

→ キーフレーム追加（ダイヤモンド アイコン）
```

**例：タイトル出現効果**
- フレーム 0: Scale 0%, Opacity 0%
- フレーム 12: Scale 100%, Opacity 100%
- → 0.5秒のズームイン効果

##### **ナレーション同期ツール**

```
Fairlight（音声編集タブ）
  ├─ Waveform 表示（波形）
  ├─ マーカー設定：ナレーション区切り
  ├─ レベル調整：-6dB ~ -3dB
  └─ 歪み検出・修正
```

```srt
1
00:00:00,000 --> 00:00:05,000
[ここにテキスト]

2

[次のテキスト]
```

```
時間: 0.25秒でフェード（24fpsの場合、6フレーム）
```

```
           │ [3秒間表示]

             └─
```

## ✅ 第4章：品質チェックリスト

### 4.1 音声品質チェック

#### **ノイズレベル**

| 評価項目 | 合格基準 | NG判定 | チェック方法 |
|---------|---------|--------|------------|
| **背景ノイズ** | 40dB以下 | 45dB以上 | Audacity → 無音部分で計測 |
| **ハム音** | 検出なし | 50-60Hz帯で-3dB以上 | Audacity → スペクトラム表示 |
| **クリップ（音割れ）** | 0件 | 1件以上 | Waveform でピーク確認 |
| **風切音** | 検出なし | 聞き取り困難 | 生耳・ヘッドフォンで試聴 |

**詳細手順**（Audacity）:

```
Step 1: 無音部分（ナレ前）選択
Step 2: Analyze → Contrast
  → Noise Floor の値を確認
  → 目安: 40dB 以下
  
Step 3: スペクトラム表示
  → Analyze → Plot Spectrum
  → 50-60Hz 帯に段ピークなし確認
```

#### **音声レベル**

| 評価項目 | 合格基準 | 説明 |
|---------|---------|------|
| **ピークレベル** | -6dB ~ -3dB | Waveform 最大値 |
| **ラウドネス** | -14 LUFS （YouTube標準） | 人間が感じる音量感 |
| **ダイナミクス** | 3-6dB の変動 | 話者の自然なボリューム変化 |

**チェック手順**（Audacity）:

```
1. Waveform（波形）表示
   - 最大ピーク: -6dB 目安
   - 小さすぎる: ノーマライズで +6dB

2. Loudness 計測
   Analyze → Loudness Normalization
   - Target: -14 LUFS
   - 実 LUFS 記録

3. ダイナミクスレンジ確認
   - 最大値 - 最小値 = 3-6dB が理想
   - 単調（差なし）= 不自然 NG
```

### 4.2 映像品質チェック

#### **解像度・フレームレート**

| 評価項目 | 合格基準 | チェック方法 |
|---------|---------|------------|
| **解像度** | 1920×1080 | 動画プロパティ確認 |
| **フレームレート** | 24fps | 動画プロパティ確認 |
| **ダラー比** | 16:9 | 画面確認 |
| **コーデック** | H.264 / H.265 | ファイル情報 |

**チェック手順（Windows）**:

```
ファイルを右クリック
  → プロパティ → 詳細
  → 解像度・フレームレート 確認

または FFmpeg コマンド:
ffprobe -v error -select_streams v:0 -show_entries \
  stream=width,height,r_frame_rate -of default=noprint_wrappers=1 \
  video.mp4

出力例:
width=1920
height=1080
r_frame_rate=24/1
```

#### **色品質**

| 評価項目 | 合格基準 | NG例 | チェック方法 |
|---------|---------|------|------------|
| **白飛び** | なし | 顔がハイキー過ぎ | Curves ヒストグラム右端 |
| **黒つぶれ** | なし | 背景が真っ黒 | Curves ヒストグラム左端 |
| **色かぶり** | なし | 全体が青っぽい | キャリブレータ確認 |
| **コントラスト** | 適度 | 平坦（のっぺり） | Curves S字カーブで補正 |

**詳細チェック**（DaVinci Resolve）:

```
[Color] ワークスペース
  → Scopes パネル（右下）
     ├─ Histogram: 左右端に棒なし確認
     ├─ Waveform: 0-100 範囲内
     ├─ Vectorscope: 円内に色分布あり
     └─ キャッシュ: スッキリ（色飽和なし）
```

#### **フォーカス・シャープネス**

| 評価項目 | 合格基準 | チェック方法 |
|---------|---------|------------|
| **ピント** | 顔が鮮鋭 | 目がはっきり |
| **モーションブラー** | なし or 自然 | 頭の動きでブレなし |
| **シャープネス** | 自然（過度でない） | エッジが硬くない |

**チェック手順**:

```
Timeline で動画を再生
  → 一時停止（顔がはっきり）
  → 確認: 目・口・輪郭がシャープ
  
NG例:
  - 目がぼけている
  - 顔全体が霧の中
  - 肌のディテールが潰れている
```

### 4.3 タイミング・シンク品質チェック

#### **ナレーション・字幕シンク**

| 評価項目 | 合格基準 | NG判定 | チェック方法 |
|---------|---------|--------|------------|
| **字幕開始** | ナレーション開始 ±0.1秒以内 | 0.2秒以上ズレ | Timeline で目視確認 |
| **字幕終了** | ナレーション終了 -0.1秒 | ナレーション終わりゴリゴリ | Timeline で目視確認 |
| **句点の同期** | 「。」で字幕終了 | 文中での終了 | 視聴テスト |

**シンク確認手順**（3回のチェック方式）:

```
チェック1: Timeline で目視確認
  - ナレーション波形（オレンジ）
  - 字幕トラック（テキスト）
  - 両者の開始・終了が一致？

チェック2: 再生しながら耳で確認
  - ナレーションを聴く
  - 字幕着目（遅延感覚なし？）
  - 違和感があれば記録

チェック3: 字幕なし再生 + 字幕あり再生を比較
  - 字幕あり視聴で「読みスピード」と「話速」が同じ？
  - タスク: 各字幕が「読める時間」を確認
```

**問題別対応表**:

| 問題 | 原因 | 対応 |
|-----|------|------|
| **字幕が遅れる** | タイムコード値が大きすぎ | 開始・終了時間を0.1-0.2秒早める |
| **字幕が先行** | タイムコード値が小さすぎ | 開始・終了時間を0.1-0.2秒遅める |
| **読む時間が短い** | 字幕表示時間が短すぎ | 終了時間を遅延 |
| **読む時間が長い** | 字幕表示時間が長すぎ | 終了時間を短縮 |

#### **アニメーション・エフェクトタイミング**

| 評価項目 | 合格基準 | チェック方法 |
|---------|---------|------------|
| **キーフレーム開始** | ナレーション「ポイント」と同期 | Timeline で確認 |
| **生成時間** | 200ms 以内 | フレーム計数 24fps換算 |
| **速度感** | 自然（急激でない） | 視聴テスト |

**エフェクト実装例**（字幕フェードイン）:

```
フレーム 0: Opacity 0%
フレーム 12: Opacity 100%
  → 0.5秒間でフェードイン
  → スムーズで自然

NG例（急激）:
フレーム 0: Opacity 0%
フレーム 1: Opacity 100%
  → 1フレーム（0.04秒）でパッと出現
  → ぎくしゃくして不自然
```
